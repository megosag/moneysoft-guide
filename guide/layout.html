# Layouts

# Einleitung

Das Programm dient zum Verarbeiten von Layouts, wie sie in Formularen und Bildschirmmasken vorkommen.

Die Hierarchie der Objekte kann zB. folgendermassen aussehen:

- EMBASSY-Programmdatenfile *.PDF
  - Maske 0
    - Zeilenart 0
      - Felder
        - Eingabefelder
        - Ausgabefelder
      - Geometrische Objekte
  - Layout 1 Variante 0
    - Zeilenart 0------------
      - Felder               \
        - Eingabefelder       \
        - Ausgabefelder        \
      - Geometrische Objekte    \
    - Zeilenart 1
      - Felder                   gleiche Struktur
        - Eingabefelder         /
        - Ausgabefelder        /
      - Geometrische Objekte  /
  - Layout 1 Variante 1      /
    - Zeilenart 0------------
      - Felder
        - Eingabefelder
        - Ausgabefelder
      - Geometrische Objekte
    - Zeilenart 1
      - Felder
        - Eingabefelder
        - Ausgabefelder
      - Geometrische Objekte
  - Layout n
    - Zeilenart 0
    - Felder
      - Eingabefelder
      - Ausgabefelder
    - Geometrische Objekte

Eine Maske ist ein Layout mit einer einzigen Zeilenart. Die Informationen von Layout und Zeilenart sind daher identisch und werden in diesem Programm unter zwei Menufunktionen angezeigt.

Layout-Varianten unterscheiden sich nur in der Varianten-Nummer und in visuellen Aspekten. Die Struktur der Zeilenarten mit gleicher Nummer muss bei allen Varianten zwangsweise die gleiche sein, da sich das Programm darauf verlässt. Dies wird von diesem Programm automatisch sichergestellt. Vor strukturellen Änderungen, die eine Programmänderung nach sich ziehen, wird vor dem Speichern gewarnt.

Eine Zeilenart besteht aus Objekten. Es gibt zwei grundsätzlich verschiedene Objektarten: Felder und Geometrische Objekte. Unter dem Begriff Grafische_Objekte sind Legenden (Beschriftungstext), Linien, Rechtecke, Ellipsen, Bilder usw. zusammengefasst. Die aufgezählten Grafischen Objekte ab "Linien" können nur auf einem grafischen Layout platziert werden.

Layouts kennt die im EMBASSY gebräuchliche Feldreihenfolge: Felder werden automatisch in der Abfolge Eingabefelder, dann Ausgabefelder gehalten. Es gibt keine Möglichkeit, ein Ausgabefeld in den Bereich der Eingabefelder zu platzieren und umgekehrt.

Layouts detektiert strukturelle Änderungen. Als Strukturelle_Änderungen gelten fundamentale Änderungen der Felderliste, zB. Felder umordnen, Feldlängen ändern usw. Bei einer Strukturellen Änderung muss das Programm angepasst werden, das das Layout benutzt. Beim Speichern einer Strukturellen Änderung wird dies durch ein Häkchen im Ausgabefeld "Struktur geändert?" angezeigt.

# Layout laden

Beim Start erscheint automatisch ein Dialog zum Laden eines Layouts. Er ist unter der Funktion "Laden" im Menu "Datei" beschrieben.

# Bearbeitung einer Zeilenart

Während der Bearbeitung einer Zeilenart erscheint zuoberst die Statuszeile. Darin werden aktuelle Daten und Zustände angezeigt:

[G0] StaMas
EMBASSY Layouts    Zeile.: r,,,,,,,,, Höhe..: h,,,,,,,,, g,,,,,                   Zeilen-Id.: i,,,,,,,,,,,,,,,,,,,,,,,,,,
a,,,,,,,,,,,,,,,,, Spalte: c,,,,,,,,, Breite: w,,,,,,,,, z,,      m,,,,,,,,,,,,,, Spalten-Id: j,,,,,,,,,,,,,,,,,,,,,,,,,,
[G0]
  • 'g': Es wird "Grid x" angezeigt, falls das Gitter aktiv ist. "x" ist der Buchstabe des gewählten Gitters.
  • 'a': Anzeige des aktuellen Vorgangs
  • 'r', 'c', 'h' und 'w': Anzeige von Koordinaten und Ausmassen oder Distanzen, abhängig vom aktuellen Vorgang.
  • 'z': Anzeige der durchschnittlichen Anzahl Zeichen im Layoutformat für die Breite 'z'
  • 'm': Anzeige des aktuellen Modus. Siehe Kapitel "Neue Objekte erzeugen". Während dem Modus Markieren wird der Name des markierten Feldes angezeigt, wenn nur ein einziges Feld markiert ist.
  • 'i', 'j': Anzeige der relevantesten Ids des zuletzt markierten Objektes. Meistens die Zeilen- und Spalten-Id.

# Fensterinhalt

Eine Zeilenart wird in einem Fenster dargestellt. Die Darstellung weicht in ein paar Details von der echten Darstellung auf dem Bildschirm oder Drucker ab.

Bei Masken werden Eingabefelder um die Ausmasse eines eventuell vorhandenen Lupenbuttons vergrössert. Erst das anzeigende Programm entscheidet ja, ob ein Lupenbutton erscheint oder nicht. Der Lupenbutton wird absichtlich nur angedeutet, damit man auch den Fall ohne Lupenbutton abschätzen kann.

Buttons werden ähnlich und in der gleichen Grösse wie ein echter dargestellt. Bei einem Textbutton (kein Flag "grafische Darstellung") kann man die Höhe daher nicht frei festlegen. Damit man den Button mit der zukünftigen Beschriftung schon sehen kann (welche ja wiederum erst das Programm festlegt) wird der Feldkommentar als Buttontext angezeigt. Falls im Kommentar einen String in Gänsefüsschen vorkommt, wird jedoch dieser zum Buttontext. Es empfiehlt sich, als Kommentar immer 'Button "Xxxx"' zu verwenden. Siehe auch "%sEmb%\TXT\Masks.TXT" Kapitel "Buttons".

Damit sich überlagernde Objekte korrekt dargestellt werden, gibt es die sog. Sichtebenen, oder mathematisch auch Z-Ebene genannt. Alle Objekte werden in verschiedenen Sichtebenen hintereinander angeordnet. Zuvorderst sind immer die Felder (gemäss aufsteigender Nummer), da sie nicht verdeckt werden dürfen. Danach folgen die grafischen Objekte. Jedes Geometrische Objekt hat als Eigenschaft eine Nummer der Z-Ebene. Je grösser diese Zahl ist, desto weiter hinten befindet sich das Objekt. Will man ein Objekt also hinter einem anderen haben, gibt man als Z- Ebene eine um 1 grössere Zahl ein. Ein Mausklick hält sich an die Z- Reihenfolge, dh. er trifft zuerst auf das Objekt, das weiter vorne liegt.

Zur Darstellung eines Objektes gibt es sog. Misch-modi, den Vordergrund-Mischmodus und den Hintergrund-Mischmodus. Am Beispiel eines Kreises ist der Vordergrund der Kreis selbst (oder die kreisförmige Fläche falls ausgefüllt). Der Hintergrund ist der Einfachheit halber bei allen Objekten rechteckig, dadurch sind šberschneidungen einfacher zu detektieren. Beim Kreis ist dies ein Rechteck, das etwas grösser ist als der Kreis selbst. Der Mischmodus bestimmt bei der Anzeige, welche Pixel bei der Anzeige dominant sind. "Quelle" bezeichet dabei Pixel des Kreises, "Ziel" die am Zielort vorhandenen Pixel. Die Pixel werden mit Boolscher Logik verknüpft. Da das ganze im Windows eigentlich nur für schwarz-weiss konzipiert wurde, muss man sich wahrscheinlich mit Durchprobieren aushelfen.

Die Zeilenart hat eine bestimmte Designgrösse und eine Sichtbare_Grösse. Beide Grössen können im Informations-Dialog der Zeilenart geändert werden. Die Designgrösse, die sog. Zeilenartausmasse, werden im Fenster durch eine horizontale und vertikale gestrichelte Linie, die Zeilenartausmass-Linien, angezeigt. Die Grenzen können per Pointer verschoben werden, einzeln oder beide zusammen. Bei einer Verschiebung wird die Sichtbare Grösse auf die neuen Ausmasse gesetzt, dh. die gesamte Zeilenart sichtbar.

Jedes Fenster hat einen sog. Einfüge-Punkt. Der Einfüge-Punkt ist der Punkt im Fenster, an dem bei bestimmten Funktionen Objekte hinkommen. Beispielsweise nimmt die Funktion "Office->Einfügen" diesen Punkt zur linken oberen Ecke der von der Zwischenablage kommenden Objekte. Der Einfüge-Punkt wird im Fenster symbolisiert durch ein Kreuz mit verkürztem linken und oberen Bein. Er ist am Anfang auf Koordinaten <0,0> und kann jederzeit neu festgelegt werden durch einen Pointerklick mit gedrückt gehaltener [Ctrl] und [Alt]-Taste. Dabei werden die neuen Koordinaten im Status angezeigt. Er kann überall hin gesetzt werden, auch ins Innere eines Objekts.

Layouts ist ein mächtiges Werkzeug. Einige Funktionen sind daher nur für Benutzer ab Privilegienstufe "Superuser" zugelassen. Betroffen sind zB. alle Funktionen zur Manipulation der Felderliste sowie Funktionen zum Verändern der Menge der vorhandenen Layouts in einem Randomfile. Weiter wird auch die Bearbeitung fundamentaler Angaben wie Feldnamen usw. in Dialogen eingeschränkt. Dadurch könnte das Funktionieren der EMBASSY-Applikationen gefährdet werden.

# Objekte markieren

  • Ein Klick auf ein unmarkiertes Objekt markiert dieses Objekt. Eine andere Markierung wird aufgehoben.

    Wenn der Pointer nicht bewegt wird, wird im Status das Ausmass- Rechteck des Objekts angezeigt. Es werden (im Gegensatz zu vorherigen Versionen von Layouts) die echten Koordinaten und Ausmasse des Objekts ohne zusätzlichen Markierungsrand angezeigt.

    Wenn nur genau ein Objekt markiert ist und es sich dabei um ein Feld handelt, wird der Name des Feldes im Modusfeld im Status angezeigt. Das dient der Orientierung, an welchem Feld man gerade ändert.

  • Aufziehen eines Rechtecks beginnend im leeren Raum markiert alle Objekte, die davon tangiert werden. Eine andere Markierung wird aufgehoben.

  • Bei gehaltener Ctrl-Taste wird der Markierzustand der betroffenen Objekte umgeschaltet (Toggle)

  • Bei gehaltener Shift-Taste werden die betroffenen Objekte der Markierung hinzugefügt (Add)

  • Ein Shift-Doppelklick auf einen Rahmen markiert seinen Inhalt dazu

  • Ein Ctrl-Doppelklick auf einen Rahmen schaltet den Markierzustand seines Inhaltes um

  • Leerschlag hebt alle Markierungen auf (Markierung aus)

  • Aufziehen eines Rechtecks im leeren Raum hebt die Markierung auf

  • Auch wenn ein Objekt durch ein anderes verdeckt ist, kann man es mit einem kleinen Trick markieren. Dazu ruft man die Funktionen "Felderliste" oder "Objekt-Liste" auf, wählt das gewünschte Objekt aus und bricht dann das Bearbeiten ab.

  • Die Zeilenartausmass-Linien (s. Kapitel "Fensterinhalt") sind auch markierbar, wenn man sie direkt anklickt. Ihre Darstellung wechselt dann auf eine durchgezogene Linie. Man kann sie auch zusammen mit anderen Objekten markieren, damit sie bei Objektverschiebungen und Vergrösserungen gleichzeitig mitverschoben werden. Beim Aufziehen eines Rechtecks werden sie jedoch nie markiert, damit sie nicht versehentlich geändert werden.

  • Einige Funktionen im Menu "Bearbeiten" dienen zur Markierung von Objekten. Insbesondere können Eingabefelder durch die Tabulator-Taste markiert werden (s. Kapitel "Eingabereihenfolge einer Maske überprüfen").

# Verschieben von Objekten

Klickt man auf ein unmarkiertes Objekt, wird dieses markiert und in den Verschiebungsmodus übergegangen. Mehrere Objekte müssen zuerst gemäss Kapitel "Objekte markieren" markiert werden.

Die markierten Objekte können per Pointer (Drag&Drop) live verschoben werden (die neue Position wird sofort sichtbar). Alternativ können sie auch mit den Cursor-Funktionen schrittweise verschoben werden. Per Tastatur sind auch grössere Verschiebungen mit den "viel"- und "Sprung"- Funktionen möglich. Man kann ein einzelnes Objekt auch exakt per Koordinaten positionieren, indem man den Informations-Dialog öffnet und die Position in Zeichen bzw. cm eingibt.

Während dem Verschieben per Pointer werden die Objekte durch eine gepunktete Linie umrahmt. In der Statuszeile wird der Eckpunkt dieses Rechtecks und die Verschiebungsdistanz vom Ausgangspunkt angezeigt.

Die Zeilenartausmass-Linien werden ebenfalls verschoben, wenn sie markiert sind. Bei einer Änderung an einer Maske wird die Maskengrösse und Sichtbare Grösse der Maske auf die neue Grösse gesetzt.

Zu Beginn des Verschiebens muss eine kleine Distanz überwunden werden bis überhaupt etwas passiert. Nachher kann frei positioniert werden. Diese Vorsichtsmassnahme verhindert, dass man durch einen Markierklick versehentlich Objekte verschiebt.

Ist beim Verschieben das Gitter eingeschaltet, wird die Verschiebung auf das Gitter gerundet, nicht etwa die Eckpunkte der Objekte. Zum Ausrichten der Eckpunkte auf das Gitter gibt es eigens die Funktion "Am Gitter ausrichten". Die Alt-Taste bewirkt, dass die Aktivierung des Gitters umgekehrt wird.

Ohne aktives Gitter wird ein Gitter von der Grösse der aktuellen Bildschirmpixel angenommen. Vorsicht: Dies ist von der momentan eingestellten Monitorauflösung abhängig und deshalb nicht zu empfehlen.

Um die markierten Objekte in den Bereich ausserhalb des sichtbaren Ausschnitts zu bringen, schiebt man sie an den Fensterrand. Wenn der Pointer in einer kleinen Zone am Rand ist, beginnt der Fensterausschnitt langsam zu rollen. Das Rollen kann beschleunigt werden, indem man den Pointer quer zur Richtung hin- und herbewegt.

Beim Ziehen der Objekte in ein anderes Fenster wird auf das systemweite Drag&Drop umgeschaltet (s. Allgemeine Anleitung). Bei diesem kann jedoch nur der Einfügeort angezeigt werden. Das Resultat kann erst beim Loslassen begutachtet werden.

# Grösse von Objekten ändern

Zuerst müssen die Objekte gemäss Kapitel "Objekte markieren" markiert werden.

Die markierten Objekte können direkt per Pointer in der Grösse geändert werden. Dazu zieht man bei einem der Objekte an einer Kante oder Ecke.

Per Tastatur sind kleine Änderungen mit den Cursor-Funktionen bei gedrückter [Shift]-Taste machbar. Es sind auch grössere Verschiebungen mit den "viel"- und "Sprung"-Funktionen (zusammen mit [Shift]-Taste) möglich.

Bei allen markierten Objekten wird immer synchron dieselbe Änderung vorgenommen. Während dem Anpassen werden die Objekte durch eine gepunktete Linie umrahmt. In der Statuszeile wird der Eckpunkt und die Grösse des umrahmenden Rechtecks angezeigt.

Tip: Die Zeilenart-Ausmass-Linien werden ebenfalls verschoben, wenn sie markiert sind.

Masken: Bei einer Grössenänderung wird auch die Sichtbare Grösse der Maske auf die neue Grösse gesetzt.

Ist beim Grösseändern das Gitter eingeschaltet, wird die Änderung auf das Gitter gerundet, nicht etwa die Grösse von Objekten. Zum Ausrichten auf das Gitter gibt es eigens die Funktion "Am Gitter ausrichten". Die Alt-Taste bewirkt, dass der Aktivierungszustand des Gitters umgekehrt wird.

Um die markierten Objekte bis in den Bereich ausserhalb des Fensters zu vergrössern, geht man mit dem Pointer bis haascharf an den Fensterrand. Wenn der Pointer in einer kleinen Zone am Rand ist, beginnt der Fensterausschnitt langsam zu rollen, wie wenn man den Pfeil im Rollbalken betätigen würde. Das Rollen kann beschleunigt werden, indem man quer zur Richtung hin- und herfährt.

Alternativ kann man ein einzelnes Objekt in der Grösse ändern, indem man den Informations-Dialog öffnet und die Ausmassdaten abändert.

Noch eine Anmerkung zu Legenden, die nicht auf das Text-Rechteck begrenzt werden: Ein Objekt hat im Layout-Editor immer nur ein einziges Ausmass-Rechteck. Das ist auch das Rechteck, das beim Resize verändert wird. Ein nichtgeclipptes (abgeschnittenes) Textstück müsste eigentlich zwei Rechtecke haben, das Clip- und das Text-Rechteck. Zwei Rechtecke sind im EMBASSY aber nicht vorhanden, weshalb der über das Clip-Rechteck herausragende Teil nicht immer korrekt aktualisiert wird. Es handelt sich dabei also nicht um einen Update-Fehler.

# Eingabereihenfolge einer Maske überprüfen

Beim Bearbeiten einer Maske werden immer wieder Felder erzeugt oder verschoben. Die Eingabereihenfolge einer Maske folgt immer den aufsteigenden Feldnummern. Die Reihenfolge kann mit den "Cursor->Tab vorwärts" und "Cursor->Tab rückwärts" überprüft werden.

Die Funktion "Tab vorwärts" ermittelt innerhalb der markierten Felder das mit der grössten Nummer und markiert das nächste Eingabefeld in der Reihenfolge. Dabei wird auch der Feldname in der Statuszeile angezeigt. Am Ende wird zyklisch auf den Anfang gesprungen. Inaktive Felder werden übersprungen.

Die Funktion "Tab rückwärts" funktioniert exakt symmetrisch in die andere Richtung.

# Das Menu "Datei"

In diesem Menu finden sich die Funktionen, die eine Zeilenart eines Layouts als gesamtes Objekt in der Datei behandeln.

# Die Funktion "Neue Zeilenart"

Diese Funktion dient zum Erzeugen einer neuen Zeilenart im aktuellen Druckformular. Der Name und Ort, wo die neue Zeilenart eingefügt wird, muss angegeben werden. Der Ort kann entweder vor einer anderen oder am Schluss sein. Lässt man das Feld "Einfügen vor" leer, wird sie am Ende angehängt. Die Bezeichnung ist nicht zwingend, sie kann auch später noch über die Funktion "Zeilenart->Informationen" festgelegt werden.

ACHTUNG: Falls die Zeilenart nicht am Ende angehängt wird, führt dies zu einer Strukturänderung des Layouts und alle Programme müssen angepasst werden. Hängt man nur am Ende an, bleibt alles kompatibel.

Hat man eine Datei mit mehreren Varianten desselben Layouts vor sich, wird die neue Zeilenart bei allen Varianten eingefügt. Sie wird per Reihenfolgenummer eingefügt. Sollte ein Variantenlayout fälschlicherweise weniger Zeilenarten enthalten und der Einfügeort hinter dem Ende liegen, erscheint eine Fehlermeldung und die Zeilenart wird am Ende angehängt.

Die Breite der neuen Zeilenart wird von der aktuellen übernommen, sofern die Koordinatenart übereinstimmt. Das Flag "Leerraum drucken" und die Papierzufuhr werden stets von der aktuellen übernommen. Defaultwerte für die Höhe und Breite sind 80*10 Standardzeichen.

Da diese Funktion tiefergehendes Wissen erfordert, ist sie nur ab Privilegienstufe "Superuser" offen.

# Die Funktion "Neue Variante erstellen"

Diese Funktion ist nur eine Kurzform der Funktion "Speichern unter". Sie setzt gerade die Daten des aktuellen Layouts ein, sodass man nur noch die gewünschte Variantennummer eingeben muss (und ev. den Dateinamen anpassen). Da die Funktion tiefergehendes Wissen voraussetzt, ist die Funktion nur ab Privilegienstufe "Superuser" offen.

# Die Funktion "Laden"

Mit dieser Funktion wird eine beliebige Zeilenart aus einem beliebigen Layout aus einer Datei geladen und ein neues Fenster zur Bearbeitung geöffnet.

Aber Achtung: Diese Funktion sollte nicht verwendet werden, um eine weitere Zeilenart aus dem gleichen Layout zu bearbeiten. Dazu muss entweder die Funktion "Weitere Zeilenart laden" verwendet werden, oder einfacher per Auswahl in der stehenden Lupe "Zeilenarten". Andernfalls ist das Layout mehrmals offen, wodurch Änderungen an Zeilenarten verlorengehen können.

Zum Laden erscheint folgende Maske:

[G0] LoaBox
File........: a...........................................................<

Layout......: b..............<   c...<   d...<   s,,,,,,,,,,,,,,,,,,,,,,,,,,,,,

Zeilenart...: e..............<   f...<                  tfo,,,,,,,,,,,,,,,,,,,

Eingabe ok?.: J<
[G0]

File ('Fil'): Eingabe eines beliebigen Randomfiles mit Masken oder Formularen drin. Die Lupe zeigt eine Liste der wahrscheinlichsten Files. Tip: Die Zusatzlupe zeigt die Liste der Files zur eingestellten Arbeitsumgebung (s. 'UWork'). Die Maske für diese Lupe kommt vom Parameter 'WorkFiles' (Funktion "Unterhalt->Konfiguration").

Layout bis Variante ('Nam')('Num')('Var'): Auswahl eines Objekts vom Typ 'SYmask' (Maske) oder 'SYform' (Formular) aus dem Randomfile. Die Lupe zeigt die vorhandenen Objekte. Nach der Auswahl wird die komplette Objekt-ID im Feld daneben angezeigt.

Zeilenart ('ParNam'): Bei einem Formular wird hier die gewünschte Zeilenart ausgewählt. Falls man per Nummer wählen möchte, gibt man hier leer ein. Falls es nur eine einzige Zeilenart gibt wird diese eingesetzt und das Feld übersprungen. Dies ist bei einer Maske immer der Fall.

Nummer ('ParNum'): Falls das Feld "Zeilenart" ausgefüllt wurde, wird hier die Nummer automatisch eingesetzt und überhüpft. Sonst kann hier die Nummer der Zeilenart angegeben werden. Die Lupe zeigt die vorhandenen Objekte. Bei einer gültigen Nummer wird der Name der Zeilenart im Feld "Zeilenart" eingesetzt.

Eingabe ok? ('Ok'): Mit "J" wird die Zeilenart bzw. Maske geladen und kann bearbeitet werden. Mit "N" wird an den Anfang gesprungen.

Mit dem Button "Test-Formular" kann direkt das Formular zum Testen der Drucker-Einzugsschächte geladen werden. Die erste Zeilenart wird sogleich geöffnet. Weiteres im Kapitel "Test-Formular für Papierzufuhr".

Nach dem Laden besteht keine fixe Beziehung zwischen der Datei und dem Fenster. Die Datei, aus der das Layout geladen wurde, ist nach wie vor frei zum Ändern, Herumschieben und Löschen.

Das Fenster für die neue Zeilenart wird automatisch so platziert, dass man möglichst viel davon sieht, aber möglichst sonst nichts verdeckt.

# Die Funktion "Weitere Zeilenart laden"

Mit dieser Funktion kann eine weitere Zeilenart aus demselben Layout wie die aktuelle geladen und bearbeitet werden. Die Funktion "Laden" sollte für diesen Zweck nicht gebraucht werden, da sonst das Layout mehrmals geladen ist.

Diese Funktion kann auch direkter ausgelöst werden, indem in der stehenden Lupe "Zeilenarten" eine Zeilenart ausgewählt wird.

# Die Funktion "Testformular laden"

Mit dieser Funktion kann direkt das Testformular für die Papierzufuhr geladen werden. Weiteres im Kapitel "Test-Formular für die Papierzufuhr".

# Die Funktion "Speichern"

Es kann nicht eine Zeilenart alleine gespeichert werden. Das Layout (Maske oder Formular) muss als ganzes abgespeichert werden. Wenn also andere Fenster auf Zeilenarten desselben Layouts offen sind, werden diese auch gleich gespeichert.

Beim Abspeichern wird ein Backup der gesamten Layout-Datei mit Erweiterung ".BAK" erzeugt. Notfalls kann man später darauf zurückgreifen.

Nach der ersten Veränderung oder falls bei der Bearbeitung strukturelle Änderungen an Feldern vorgenommen wurden, wird vor dem Speichern mit folgender Maske nachgefragt, was zusätzlich noch gemacht werden soll (man kann die Maske auch Erzwingen durch gedrückthalten von [Shift] während der Auslösung der Funktion). Bricht man hier ab, kehrt man zum Zustand vor Auslösung der Speicherung zurück.

[G0] MatVar
Layout.................: A,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,
                         AI,,,,,,,,,,,,,,,,,,,,,,,,,,,,

Struktur geändert?.....: B>

Ausmasse ZA überprüfen?: f<
Ins PDF übertragen?....: a< b...........................................................<_
Varianten anpassen?....: c< Anzahl: C,,,
Deklaration anpassen?..: d< e...........................................................<

Eingabe ok?............: J<
[G0]

Layout: Zeigt die Beschreibung des Layouts und dessen Objekt-Id an.

Struktur geändert: Zeigt an, ob seit dem letzten Speichern strukturelle Änderungen vorgenommen wurden. Falls ja, muss auch das zugehörige Programm angepasst werden, sonst kann es Stops geben.

Ausmasse ZA überprüfen? ('Ext'): Dieses Feld ist nur bei grafischen Formularen eingebbar. Bei "Ja" wird die Funktion "Ausmasse überprüfen" im Menu "Layout" aufgerufen. Bricht man einen dort erscheinenden Dialog ab, wird das gesamte Speichern abgebrochen.

Ins PDF übertragen? ('Pdf'): Eingabe, ob man das Layout nach dem Speichern gleich ins Programmdatenfile übertragen möchte. Das geht zur Sicherheit nur, wenn das Layout bereits im PDF vorhanden ist (sonst geht ev. der Eintrag FCO-File vergessen und beim nächsten Zusammenbau fehlt das Layout). Falls das aktuelle Layout also aus einem PDF geladen wurde, wird das Feld beim ersten Erscheinen der Maske auf "Nein" vordefiniert. Auch bei Eingabe von "Nein" bleibt das Sourcefile im nachfolgenden Feld erhalten, damit man leicht umschalten kann. Der Zustand dieses Feldes wird pro Fenster aufbewahrt. Das Feld kann auch mit Ctrl-P umgeschaltet werden.

Programmdatenfile ('PdfFil'): Eingabe des Filenamens des Programmdatenfiles, in das das Layout übertragen werden soll. Das Layout muss dort bereits existieren. Das File darf nicht dasselbe sein, wie das, aus dem das Layout geladen wurde. Falls der Filename des aktuellen Layouts der EMBASSY-Konvention entspricht, wird beim ersten Mal als Vorschlag das Programmdatenfile eingesetzt, wie es der Namenssystematik entspricht. Der Inhalt dieses Feldes wird pro Fenster aufbewahrt. Das Feld kann nur eingegeben werden, wenn man im linken Feld "Ja" gesagt hat.

Varianten anpassen? ('Ada'): Eingabe, ob man die zusätzlichen Varianten (die Anzahl ist rechts angezeigt) strukturell anpassen will. Es gibt nur sehr selten den Fall, dass man hier Nein sagt. Bei Layouts ohne Zusatzvarianten ist dieses Feld immer abgeschaltet und wird übersprungen. Dasselbe auch, wenn die Struktur nicht geändert wurde. Das Feld kann auch mit Ctrl-V umgeschaltet werden.

Beim Anpassen_der_Varianten wird die Feldstruktur des aktuellen Layouts (nicht etwa der Variante 0) zur Basis. Alle anderen Varianten werden daran angepasst, wobei die visuellen Daten der Felder (Ausmasse, Formate usw.) erhalten bleiben. Bei Datums- und Zeitfeldern bleibt ausnahmsweise auch die Länge erhalten, da sie mit dem Format im Zusammenhang steht und im Programmcode keine Veränderung bewirkt.

Es werden alle Zeilenarten der beiden Layouts verglichen. Falls die Zeilenarten nicht zusammenpassen erscheint eine Fehlermeldung und die Zeilenart bleibt unverändert. Wenn die Zeilenart gefunden wurde, werden die Felder abgeglichen. Die šbereinstimmung von Feldern erfolgt per Feldname. šberschüssige Felder in den Varianten werden gelöscht.

Deklaration anpassen? ('Dcl'): Eingabe, ob man die Deklarationen in einem Sourcefile nachführen will. Normalerweise muss dies bei strukturellen Änderungen immer gemacht werden. Auch bei Eingabe von "Nein" bleibt das Sourcefile im nachfolgenden Feld erhalten, damit man leicht umschalten kann. Der Zustand dieses Feldes wird pro Fenster aufbewahrt. Das Feld kann auch mit Ctrl-D umgeschaltet werden.

Sourcefile ('DclFil'): Eingabe des Filenamens des Sourcefiles, in dem die Deklarationen angepasst werden sollen. Das Feld kann nur eingegeben werden, wenn man links "Ja" gesagt hat. Falls der Filename des Layouts der EMBASSY-Konvention entspricht, wird als Vorschlag das Sourcefile eingesetzt, wie es der Namenssystematik entspricht. Andernfalls wird das Sourcefile der Arbeitsumgebung (s. 'UWork') vorgeschlagen. Der Inhalt dieses Feldes wird pro offene Zeilenart aufbewahrt.

Das Deklarieren_eines_Layouts im Sourcefile geschieht durch Fernsteuerung von 'UDeclare'. Wenn das Sourcefile in einer anderen Session im Editor offen ist, muss man nachher in dieser Session die Funktion "Abgelegtes Programm" im Standardmenu "Kontrolle" aufrufen, um die Deklarierung echt durchzuführen (s. Handbuch von 'UDeclare').

Zum Deklarieren eines Layouts zu einem anderen Zeitpunkt als dem Speichern rufe man 'UDeclare' direkt über den Toolverteiler 'UProgs' auf.

Vor dem Speichern wird geprüft, ob das zu überschreibende Layout am Zielort bereits durch einen anderen Prozess verändert wurde. Falls ja, wird mit dem folgenden Dialog nachgefragt, ob trotzdem gespeichert werden soll:

A,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,,

und damit das des anderen Benutzers

[G0]

Das ganze stellt sicher, dass sich nicht zwei Benutzer versehentlich gegenseitig in die Quere kommen.

# Die Funktion "Speichern unter"

Mit dieser Funktion kann das aktuelle Layout in ein anderes File unter einer anderen Objekt-Id abgespeichert werden.

[G0] SpeBox
File........: a........................................................<
  neu?......: A>  (Ursprungsfile wird dupliziert)

Layout......: N,,,,,,,,,,,,,,   c...<   d...<   S,,,,,,,,,,,,,,,,,,,,,,,,,,,,,

Eingabe ok?.: J<
[G0]

File ('Fil'): Eingabe des Filenamens des Zielfiles. Als Default wird der Filename eingesetzt, woraus das Layout geladen wurde. Falls es noch kein File diesen Namens gibt, wird beim Speichern zuerst ein Duplikat des Files gemacht, aus dem die aktuelle Zeilenart geladen wurde. Dies wird im Ausgabefeld unterhalb angezeigt. Falls das File existieren sollte, muss es sich im ein Randomfile handeln.

Layout ('Num')('Var'): Eingabe einer neuen Nummer und/oder Variantennummer für das zu speichernde Layout. Als Default werden die Nummern des aktuellen Layouts eingesetzt. Der Name des Layouts wird aus den Layout-Informationen genommen und kann nicht geändert werden. Wird nur die Variantennummer geändert, kann so eine neue Variante eines Layouts erstellt werden. Dafür gibt es jedoch die Funktion "Neue Variante erstellen".

Das Ausgabefeld "neu?" zeigt an, ob es noch keine Datei mit dem angegeben Namen gibt. Da die Erzeugung eines Layouts sehr kompliziert ist, wird in diesem Fall einfach die bestehende Datei kopiert. Falls es sich beim Zielfile um ein TDF oder GDF handelt (welches ja per Definition nur ein einzelnes Layout enthalten soll) werden alle Layouts mit anderer Nummer oder Variante (abgesehen von Sprachvarianten) aus der Kopie entfernt.

Da die Funktion tiefergehendes Wissen voraussetzt, ist die Funktion nur ab Privilegienstufe "Superuser" offen. Am Ende der Funktion sieht das aktuelle Layout so aus, als ob es aus dem Zielfile geladen wurde.

# Die Funktion "Alle Speichern"

Die Funktion speichert alle Layouts mit der Funktion "Speichern", aber ohne vorher nachzufragen. Falls das Speichern eines Layouts abgebrochen wird, wird die Funktion als Ganzes abgebrochen.

# Die Funktion "Schliessen"

Diese Funktion schliesst das aktuelle Fenster. Falls es sich um das letzte Fenster auf die Datei handelt, wird auch die Datei geschlossen. Bei Änderungen an der Datei wird nachgefragt, ob gespeichert werden soll.

# Die Funktion "Alle Schliessen"

Diese Funktion schliesst alle offenen Fenster mit der Funktion "Schliessen". Vor dem Start der Funktion wird zur Sicherheit noch nachgefragt. Dabei kann mit dem Button "Alle speichern" bewirkt werden, dass die veränderten Layouts beim Schliessen automatisch ohne Nachfrage gespeichert werden.

# Die Funktion "Freigeben"

Diese Funktion ist eine schnelle Methode, das Bearbeiten einer geöffneten Datei zu Beenden und die Datei zu schliessen. Das kann nützlich sein, wenn man die Datei verschieben will.

Es werden alle Fenster auf die Datei geschlossen, deren Fenster im moment gerade aktiv ist. Falls etwas geändert wurde wird nachgefragt, ob man die Änderungen speichern will.

Vor dem Start der Funktion wird zur Sicherheit noch nachgefragt.

Die Funktion ist auch durch die Funktion "Löschmodus" aufrufbar.

# Die Funktion "Löschen" (Datei)

Diese Funktion dient zum Löschen des geöffneten Layouts. Nach einer Nachfrage werden alle offenen Fenster auf das Layout geschlossen und das Layout aus der Datei herausgelöscht. Vor dem Löschen wird zur Sicherheit eine Kopie der Datei mit der Endung ".BAK" erstellt.

Drucker ('Pri'): Auswahl des grafischen Druckers im Betriebssystem, auf den gedruckt werden soll. Im Windows müssen neue Drucker vorher im Kontrollfeld "Drucker" eingerichtet werden. Der Name "Standard" bezeichnet denjenigen Drucker, der beim Betriebssystem als Standarddrucker markiert wurde. Unter dem Feld wird eine Beschreibung des Druckers angezeigt. In allen anderen Fällen werden die Werte (A4 hoch, Ränder usw.) aus den momentanen Druckeinstellungen des Druckers genommen. Wenn man einen textuellen Ausdruck, oder einen Ausdruck mit speziellen Einstellungen machen will, muss dieses Feld leergelassen werden und dafür im nachfolgenden Feld eine Steuerung eingegeben werden.

Steuerung ('Prp'): Auswahl einer Drucksteuerung für textuelle Drucke oder Drucke mit speziellen Einstellungen (Setup). Weiteres siehe Allgemeine Anleitung. Das Feld kann nur eingegeben werden, wenn das Feld "Drucker" leergelassen wurde.

Falls ein textuelles Layout grafisch gedruckt werden soll, wird es zuerst auf grafisch umgewandelt. Danach werden alle angegebenen Zeilenarten der Reihe nach in der angegebenen Anzahl untereinander ausgedruckt.

# Die Funktion "LIS-File schreiben"

Mit dieser Funktion können die Felderliste aller Zeilenarten des aktuellen Layouts in ein LIS-File geschrieben werden. Ein LIS-File dient zur textuellen Darstellung der Felderliste eines Layouts. Das LIS- File dient heutzutage nur noch Dokumentations-Zwecken. Wenn jedoch auch eine textuelle Geometrie dazu vorhanden ist, kann durch sie mithilfe von 'ULayouts' wieder ein Layout (mit Zeichenkoordinaten) erzeugt werden.

Als Filename wird gleich der Standard eingesetzt.

Der Felderliste wird eine leere Geometrie vorangestellt, und die Kolonne 'id' der Felderliste wird leer generiert.

Nach dem Schreiben ins File wird sogleich ein Edit darauf geöffnet, damit man die Liste sofort ansehen und ev. ausdrucken kann.

# Das Menu "Bearbeiten"

Die Befehle aus dem Menu "Bearbeiten" und dem Menu "Office" dienen zum kopieren, löschen und einfügen von Objekten. Mithilfe der Zwischenablage können Objekte auch mit anderen Applikationen ausgetauscht werden.

# Die Funktion "Löschen" (Bearbeiten)

Diese Funktion löscht Felder, Legenden oder Geometrische Objekte aus der Liste. Die Zwischenablage bleibt unverändert.

Die Funktion wird auch durch die Taste "Delete" (Funktion "Zeichen entfernen") ausgelöst.

# Die Funktion "Sichtbar <-> Unsichtbar"

Die Funktion macht Felder und Geometrischen Objekte sichtbar oder unsichtbar. Alles andere an den Objekten bleibt unverändert.

Bei Feldern wird das Flag "Aktiv/Sichtbar" umgeschaltet. Ein inaktives Feld auf einer Maske wird bei der Dialog-Eingabe überhüpft.

Bei Grafischen Objekten wird das Flag "Unsichtbar" umgeschaltet.

Unsichtbare_Objekte sind in allen Belangen vollwertige Objekte nur mit dem Unterschied, dass sie bei der echten Anwendung des Layouts nicht dargestellt werden. Damit sie jedoch im Layout-Editor gesehen und bearbeitet werden können, werden sie durch eine gepunktete Umrandungs- linie angezeigt und können wie sichtbare Objekte markiert werden.

Das Unsichtbarmachen von Objekten kann etwas schneller auch mit der Taste "<-" (Backspace, Funktion "Zeichen löschen") ausgelöst werden.

# Die Funktion "Office->Kopieren"

Die Funktion befindet sich im Standardmenu "Office".

Alle markierten Objekte werden als Objekt-Liste in die Zwischenablage kopiert. Auch Felder sind darin eingeschlossen.

Falls sich Legenden, Bitmaps und MetaFiles in der Markierung befinden, werden diese auch abgelegt. Die Zwischenablage kann maximal ein Objekt pro Format speichern. Nur die letzten Objekte jeden Typs bleiben daher gespeichert.

# Die Funktion "Office->Ausschneiden"

Die Funktion befindet sich im Standardmenu "Office".

Es handelt sich um die Zusammensetzung von "Kopieren" und "Löschen".

# Die Funktion "Office->Einfügen"

Die Funktion befindet sich im Standardmenu "Office".

Die Funktion fügt eine Objektliste, einen unformatierten Text, ein MetaFile oder eine Bitmap am Einfüge-Punkt ein. Diese Formate werden in genau dieser Reihenfolge aus der Zwischenablage herausgesucht. Im folgenden wird das Einfügen einer Liste von Objekten beschrieben, die vorher einmal von diesem Programm mit der Funktion "Office->Kopieren" in die Zwischenablage gelegt wurde. Das Einfügen eines bestimmten Formates kann in der Beschreibung zum Menu "Einfügen" nachgelesen werden.

Beim "Einfügen" eines Feldes wird nachgeschaut, ob es bereits ein Feld gleichen Namens gibt. Falls ja, wird am Ende die Laufnummer 0 angehängt oder die Laufnummer um 1 erhöht, falls bereits eine enthalten ist. Die neuen Felder werden am Ende des gleichen Typs (Ein/Ausgabefeld) angehängt.

Da Felder nicht immer einfach so erzeugt werden sollten, gibt es noch zwei Varianten dieser Funktion:

  1. Falls man die [Shift]-Taste gedrückt hält beim Auslösen der Funktion wird nach einem Feld gleichen Namens gesucht. Wenn eines vorhanden ist, wird nicht ein neues Feld erzeugt, sondern nur die Informationen auf das vorhandene Feld übertragen. Wird keines mit gleichem Namen gefunden, wird das Feld normal eingefügt.

  2. Falls man die [Shift]- und [Alt]-Taste gedrückt hält beim Auslösen der Funktion geschieht dasselbe wie bei 1), jedoch werden die Feldinformationen jetzt per Feldnummer anstatt per Feldname übernommen. Das ist insbesondere bei Zeilenarten mit gleicher Struktur aber anderen Feldnamen, zB. mehreren Sprachvarianten, nützlich.

Falls während dem Einfügen ein Gitter aktiv ist, werden die eingefügten Objekte zuerst mit der Funktion "Position auf Gitter runden" auf Gitterkoordinaten geschoben und danach die Ausmasse per Funktion "Ausmasse auf Gitter" auf Gitterausmasse gerundet. Will man das nicht haben, muss man das Gitter abschalten oder das Einfügen mit gedrückt gehaltener [Alt]-Taste auslösen. Letzeres schaltet wie beim Objektverschieben den Zustand des Gitters für die Ausführung der Funktion um.

Am Schluss wird der Einfügepunkt unter die eingefügten Objekte platziert, damit sofort weiter eingefügt werden kann.

Hinweis: Die Koordinatenart in der Zwischenablage muss mit der momentanen Koordinatenart übereinstimmen, sonst erscheint eine Fehlermeldung. Das liegt daran, dass die Platzierung der Objekte im Gitter nur unter Kenntnis aller Layout-Informationen möglich ist. Man könnte aber Objekte aus einer textuellen Zeilenart übernehmen, indem man die Zeilenart im Speicher mit der Funktion "Koordinatenart ändern" auf Units konvertiert, kopiert und wieder einen Schritt zurück geht.

Ganz neue Objekte erzeugt man nicht mit dieser Funktion, sondern mit den Funktionen im Menu "Einfügen" oder "Modus".

# Die Funktion "Duplizieren"

Mit dieser Funktion kann man eine Kopie der markierten Objekte machen. Sie werden etwas versetzt nach unten eingefügt und bilden die neue Markierung. Bei eingeschaltetem Gitter werden die Objekte um eine Masche nach unten versetzt angeordnet, sonst um 1 cm.

Bei Kopieren von Feldern werden deren Namen umnummeriert falls nötig. Dabei wird am Ende die Laufnummer 0 angehängt oder dessen Laufnummer um 1 erhöht, falls bereits eine enthalten ist.

Eine andere Möglichkeit ist, dass man sie mit der Funktion "Office:Kopieren" in die Zwischenablage kopiert und danach mit "Office:Einfügen" wieder einfügt. Diese Funktion macht eigentlich genau das, ausser dass duplizierte Felder direkt hinter dem Original in die Felderliste eingefügt werden.

Die Funktion kann auch durch die Funktion "Felderst kopieren" ausgelöst werden.

# Die Funktion "Rahmeninhalt markieren"

Durch diese Funktion werden alle Objekte innerhalb der markierten Rahmen zur Markierung hinzugenommen. Andere Objekttypen in der Markierung werden ignoriert. Die Funktion kann auch durch einen [SHIFT]-Doppelklick auf den Rahmen ausgelöst werden.

Bei gedrückt gehaltener [Ctrl]-Taste wird der Markierzustand des Rahmeninhaltes umgekehrt anstatt eingeschaltet. Das gleiche kann durch einen [Ctrl]-Doppelklick auf den Rahmen bewirkt werden.

# Die Funktion "Alles markieren"

Es werden alle Objekte der Zeilenart markiert.

# Die Funktion "Informationen"

Die Funktion startet den Informations-Dialog, je 1 Mal für jedes markierte Objekt. Falls ein Dialog abgebrochen wird, endet die Funktion.

# Die Funktion "In Eingabefeld umwandeln"

Die Funktion kann nur auf Masken verwendet werden. Auf Formularen ist sie immer inaktiv.

Mit dieser Funktion werden alle markierten Ausgabefelder in Eingabefelder umgewandelt. Dazu wird das Flag "Ausgabe-Feld" abgeschaltet und das Feldformat auf das vom Grundformat der Zeilenart abgeleitete Eingabefeldformat gesetzt. Sodann wird das Feld ans Ende der Eingabefelder verschoben.

Andere Objekte in der Markierung werden ignoriert.

# Die Funktion "In Ausgabefeld umwandeln"

Mit dieser Funktion werden alle markierten Eingabefelder in Ausgabefelder umgewandelt. Dazu wird das Flag "Ausgabe-Feld" eingeschaltet und das Feldformat auf das vom Grundformat der Zeilenart abgeleitete Ausgabefeldformat gesetzt. Sodann wird das Feld ans Ende der Eingabefelder verschoben.

Andere Objekte in der Markierung werden ignoriert.

# Die Funktionen "Zurück" und "Vorwärts"

Mit diesen Funktionen kann man Änderungen an der Zeilenart schrittweise rückgängig machen. Falls man zu weit zurück gegangen sein sollte, kann man auch wieder vorwärts inklusive dem Stand vor dem ersten Rückschritt.

Es können nahezu alle Operationen rückgängig gemacht werden, da die gesamte Zeilenart vor der Operation wegkopiert wird. Veränderungen an Layout-Informationen (Masken-Info, Formular-Info) können nie rückgängig gemacht werden. Bei grossen nicht rückgängig machbaren Operationen (zB. Koordinatenart wechseln) wird vorher in einem Dialog gewarnt.

Layouts verfügt im Moment über 20 gespeicherte Zurückhol-Zustände pro Fenster. Durch Neukompilation könnte dies falls nötig auch erhöht werden, braucht aber ev. viel Speicherplatz, wenn das Layout sehr gross ist.

# Die Funktion "Informationen"

Mit dieser Funktion kann der Info-Dialog aller markierten Objekte angezeigt werden. Es wird ein Info-Dialog nach dem anderen geöffnet, sofern man im Dialog nicht abbricht. Dadurch kann man die Einstellungen mehrerer Objekte in einem einzigen Durchlauf anpassen.

Der Informations-Dialog eines Objektes kann direkter auch durch einen Doppelklick auf das Objekt aufgerufen werden.

Speziell zu erwähnen am Feld-Informations-Dialog: Falls man beim Einfügen das Flag "Ausgabefeld" umschaltet, wird nach der Bearbeitung automatisch die Funktion "In Eingabefeld umwandeln" bzw. "In Ausgabefeld umwandeln" aktiviert. Dadurch bleibt die Feldreihenfolge immer erhalten und das Feldformat muss nicht von Hand angepasst werden.

# Die Funktion "Format ändern"

Mit dieser Funktion kann man auf einfache Weise das Format von mehreren markierten Objekten jegwelchen Typs auf ein bestimmtes setzen. Das Format wird per Lupe ausgewählt. Die Lupe wird auf das Format vorpositioniert, das alle Objekte im Moment haben. Falls die Objekte unterschiedliche haben sollten, wird auf das Grundformat des Layouts vorpositioniert.

# Die Funktion "Position auf Gitter runden"

Diese Funktion bezieht sich auf die eingestellten Gitterwerte. Sie korrigiert alle Koordinatenangaben der markierten Objekte, indem sie sie auf den nächstliegenden Gitterpunkt rundet. Ausmasse (Höhe, Breite) werden dabei nie verändert.

Bei Legenden ist die "Position" der Ausgangspunkt des Textstücks, dh. bei einem rechtsbündingen Text wird die Endkoordinate auf das Gitter gerundet.

Bei Linien wird sowohl der Anfangs- wie der Endpunkt auf die Mitte einer Gittermasche positioniert.

Bei Rechtecken und Ellipsen wird die linke obere Ecke auf die Mitte einer Gittermasche positioniert.

Es wird auch ausgerichtet, wenn das Gitter im Moment nicht aktiv ist. Es wird jedoch nur diejenige Ordinate ausgerichtet, für die im Dialog "Gitter setzen" eine Maschenweite angegeben wurde.

# Die Funktion "Breite auf Gitter runden"

Diese Funktion verwendet die eingestellten Gitterwerte. Sie korrigiert alle Breiten der markierten Objekte, indem sie sie auf den nächstliegenden Gitterpunkt rundet. Die Position wird dabei nie verändert.

Bei Legenden wird die Breite ausgehend vom Ausgangspunkt des Textstücks korrigiert, dh. bei einem rechtsbündingen Text wird die Anfangsordinate verschoben.

Es wird auch ausgerichtet, wenn das Gitter im Moment nicht aktiv ist. Es wird jedoch nur dann ausgerichtet, wenn im Dialog "Gitter setzen" die horizontale Maschenweite nicht leer ist.

# Die Funktion "Ausmasse auf Standard"

Die Funktion wirkt auf alle Felder und Legenden, die markiert sind. Falls die Zeilenart-Ausmasslinien markiert sind, werden am Ende aller Anpassungen auch die Zeilenart-Ausmasse neu berechnet. Andere Objekttypen werden ignoriert.

Die Funktion berechnet bei Legenden die exakten Ausmasse (das "Bounding rectangle") mithilfe des Objektformats neu und setzt sie beim Objektrechteck ein.

Bei einem Feld wird die exakte Höhe sowie die Standardbreite für die momentane Feldlänge mithilfe des Objektformats berechnet und eingesetzt. Die Breite wird aber noch auf maxmimal 60 Zellen begrenzt.

Bei Bildbuttons (Flag "graphisch" eingeschaltet) gibt es keine Standardgrösse. Die Grösse des Bildes ist nur dem Programmierer bekannt, weshalb diese Funktion die Werte unberührt lässt.

Das Gitter übt auf diese Funktion keinen Einfluss aus.

# Die Funktion "Eingabefelder einordnen"

Die Funktion dient zum Wiederherstellen der Eingabereihenfolge, nachdem auf einer Maske Eingabefelder umgeordnet wurden. Die Eingabe der Felder im EMBASSY erfolgt in aufsteigernder Feldnummer. Nach dem Umordnen von Feldern auf der Maske müssen also auch die Feldnummern neu vergeben werden. Dazu markiert man einen Bereich von Eingabefeldern (ev. inkl. Ausgabefelder und Legenden) und ruft diese Funktion auf.

Die Funktion ermittelt zuerst einmal, welche Eingabefelder markiert sind. Alle anderen Objekte werden ignoriert. Sodann werden die Felder geometrisch geordnet und durchnummeriert, zuerst von links nach rechts, dann die darunter folgenden Felder. Das Prinzip ist etwa das gleiche, wie wenn man einen Textabsatz liest. Nach der Sortierung erscheint eine Lupe zur Auswahl des Eingabefeldes, vor dem der Block eingeschoben werden soll. Der unterste Eintrag symbolisiert das Ende der Liste. Die Lupe wird auf das erste nichtmarkierte Feld hinter dem ersten zu einzuordnenden Feld vorpositioniert.

Man kann mit dieser Funktion natürlich auch nur ein einziges Eingabefeld an einen anderen Ort in der Felderliste verschieben.

Der Aufbau der Felderliste kann jederzeit mit der Funktion "Zeilenart->Felderliste" angezeigt werden. Die Eingabereihenfolge kann ebenfalls auf einfache Weise überprüft werden, siehe dazu das Kapitel "Eingabereihenfolge einer Maske überprüfen".

# Die Funktion "Ausgabefelder einordnen"

Diese Funktion macht dasselbe wie die Funktion "Eingabefelder einordnen", jedoch für Ausgabefelder. Sie dient zum Herstellen einer bestimmten Reihenfolge der Ausgabefelder, damit man sie zB. in einem Zug clearen kann.

# Die Funktion "Zu Multifeld kombinieren"

Mit dieser Funktion kann aus mehreren einzelnen Feldern ein grosses Multifeld erzeugt werden.

Ein Multifeld besteht aus einem oder mehreren Alpha-Feldern (Ein- oder Ausgabe möglich). Das erste Feld wird mit dem Flag "1. Multifeld" markiert und gibt die Breite des resultierenden Feldes an. Alle weiteren Felder dienen nur zur Aufnahme von weiteren Zeichen. Sie haben das Flag "2./weiteres Multifeld" und sind nicht aktiv.

Diese Funktion schaltet die Flags der markierten Felder entsprechend um und vergrössert die Höhe des ersten Feldes so, dass die Unterkante auf der Unterkante des untersten markierten Feldes zu liegen kommt. Das erste Feld überdeckt dann die dahinterliegenden inaktiven Folgefelder. Falls die Felder vorher schon Multiflags haben werden sie umgesetzt. Man darf also in einem textuellen Layout bereits diese Flags zur besseren Visualisierung setzen.

Falls die markierten Felder aus einem der obengenannten Gründen nicht zu einem Multifeld kombiniert werden können, erscheint eine Fehlermeldung.

# Die Funktion "Multifeld zerlegen"

Diese Funktion wirkt auf alle Multifelder in der Markierung. Andere Objekte werden ignoriert.

Bei jedem Multifeld werden die nachfolgenden Felder ermittelt und dann die Flags umgeschaltet. Sodann wird die Höhe des ersten Feldes wieder auf die Standardhöhe gesetzt (wie bei der Funktion "Ausmasse auf Standard").

# Die Funktion "Makro auf mark. Objekten"

Diese Funktion merkt sich, welche Objekte markiert wurden. Sodann markiert sie jedes Objekt einzeln und führt das vorher vom Benutzer mit der Funktion "Weiteres->Dialog->Macro aufzeichnen" aufgezeichnete Makro aus (s. Allgemeine Anleitung). Das Makro muss daher exakt alle Aktionen ab dem Zeitpunkt enthalten, ab dem eine einzelnes Objekt markiert wurde.

Diese Funktion ist nicht zu unterschätzen. Sie ist extrem mächtig und man kann mit ihr jede Funktionalität erreichen, die im Layouts nicht bereits vorprogrammiert ist!

# Das Menu "Einfügen"

In diesem Menu befinden sich die Funktionen zum Einfügen von Objekten in die aktuelle Zeilenart.

Eingefügte Objekte übernehmen möglichst viele Informationen vom Ausgangsort. Fehlende Informationen werden zT. ergänzt mithilfe der Eigenschaften, die in der Funktion "Default-Werte" eingestellt sind.

Ohne Koordinatenvorgabe wird ein Objekt dort eingefügt, wo der Einfüge- Punkt ist. Danach wird wird der Einfügepunkt unter die eingefügten Objekte platziert, damit sofort weiter eingefügt werden kann.

Felder ohne Reihenfolgenvorgabe werden in der Felderliste dort eingefügt, wo es mittels der Funktionen "Vor markiertem Feld" bzw. "Am Schluss" gesagt wurde.

Die folgenden Objekttypen können aus einem anderen Windows-Fenster per Drag&Drop eingefügt werden:

  • Eine Menge von Objekten (aus anderen Layouts-Fenstern)
  • Bitmaps
  • MetaFiles
  • Text Es wird genau gleich eingefügt, wie über die Zwischenablage.

Am Schluss sind jeweils die eingefügten Objekte markiert.

# Die Funktion "Rahmen um mark. Objekte"

Mit dieser Funktion kann auf einfache Weise ein Rahmen um die markierten Objekte eingefügt werden. Die Funktion ist eine bequemere Form des Modus "Rahmen/Trennlinie def.".

Die Funktion berechnet die Begrenzungslinie um die markierten Objekte und legt einen Rahmen darum. Ob ein Gitter aktiv ist, ist unerheblich. Im folgenden Rahmendialog kann der Titel und die anderen Angaben festgelegt werden.

Am Schluss sind der Rahmen und alle darin befindlichen Objekte markiert. Das ermöglicht ein sofortiges Verschieben falls nötig.

# Die Funktion "Legende zu mark. Feldern"

Diese Funktion generiert zu allen markierten Feldern eine Legende. Als Text wird der Feldkommentar genommen.

Die Legende hat immer einen Doppelpunkt am Ende und bekommt die Ausrichtung "Expandieren". Die Legende endet gemäss Standard 1 Zellenbreite vor dem Feld. Die Breite wird bestimmt durch Abschauen bei weiter oben liegenden Legenden, die am selben Ort enden und ebenfalls expandiert sind. Wenn nichts zu Abschauen gefunden wird, wird die Legende 20 Zellen breit. Das Format wird gemäss Feldtyp vom Grundformat abgeleitet. Der Mischmodus wird aus den Default-Werten genommen.

Die Legende wird nur generiert, wenn mind. 1 Zeichen Platz hat. Wenn weniger Platz als nötig zur Verfügung steht, wird sie gekürzt.

Das Legendenobjekt wird am Ende der Objektliste angehängt.

# Die Funktion "Text aus Zwischenablage"

Diese Funktion fügt den unformatierten Text als Legende in die Zeilenart ein, der sich im Moment in der Zwischenablage befindet. Jede Zeile wird zu einer linksbündigen Legende, alle untereinander angeordnet.

Anders als "Office->Einfügen" wählt diese Funktion gezielt nur das Textformat aus, auch wenn andere Formate in der Zwischenablage zur Verfügung stehen würden.

# Die Funktion "MetaFile aus Zwischenablage"

Diese Funktion fügt das MetaFile in die Zeilenart ein, das sich im Moment in der Zwischenablage befindet. Das MetaFile wird am Einfüge- Punkt eingefügt.

Anders als "Office->Einfügen" wählt diese Funktion gezielt nur das MetaFile aus, auch wenn andere Formate zur Verfügung stehen würden.

# Die Funktion "Bitmap aus Zwischenablage"

Diese Funktion fügt die Bitmap in die Zeilenart ein, die sich im Moment in der Zwischenablage befindet. Dabei wird der EMBASSY-Standarddialog zum Bearbeiten von Bitmap-Informationen aufgerufen. Die Bitmap wird am aktuellen Einfüge-Punkt eingefügt.

Anders als "Office->Einfügen" wählt diese Funktion gezielt nur die Bitmap aus, auch wenn andere Formate zur Verfügung stehen würden.

# Die Funktion "Bitmap aus Datei"

Diese Funktion lädt eine Bitmap aus einer Datei und fügt sie in die Zeilenart ein. Dabei wird der EMBASSY-Standarddialog zum Bearbeiten von Bitmap-Informationen aufgerufen. Die Bitmap wird am aktuellen Einfüge- Punkt eingefügt.

Es können alle Bitmaps vom Typ "Windows Bitmap" gelesen werden. Normalerweise haben solche Dateien die Endung ".BMP".

# Die Funktion "Attr-/Feldliste öffnen"

Mit dieser Funktion kann eine neue Stehende Lupe mit Feldern aus einer Feldliste oder Attributen aus einer Attributsliste geöffnet werden. Wenn eine Maske angegeben wird, wird automatisch deren Felderliste geöffnet. Analog wird bei Angabe einer Zeilenart aus einem Formular automatisch deren Felderliste geöffnet. Der Koordinatentyp der Zeilenart muss nicht zwingend mit der aktuellen übereinstimmen.

Bei der Auswahl einer Attributsliste wird daraus eine Feldliste konstruiert. Zur Ergänzung der fehlenden Angaben wie zB. Formate wird auf die Einstellung unter der Funktion "Default-Werte" zurückgegriffen.

Die Stehende Lupe bleibt von nun ab offen. Sie bleibt auch offen, wenn alle Zeilenarten-Fenster geschlossen werden. Aus der Lupe können Felder an den Einfüge-Punkt des zuletzt aktiven Fensters eingefügt werden, indem man auf einen Eintrag doppelklickt. Auf ein Formular wird ein Eingabefeld automatisch als Ausgabefeld platziert. Mit [Ctrl]- Doppelklick kann man erreichen, dass ein Eingabefeld als Ausgabefeld bzw. ein Ausgabefeld als Eingabefeld eingefügt wird.

Die linke obere Feldecke wird an den Einfüge-Punkt gelegt. Als Ausnahme wird der Einfügepunkt vorher 21 Zeichen nach rechts gerückt, wenn er sich noch auf <0,0> befindet. Das ist eine Erleichterung zu Beginn eines neuen Layouts. Es ergibt sich eine 20 Zeichen breite Legende.

Zur Reihenfolge: Das neue Feld wird vor dem Einfügefeld in die Felderliste eingefügt (s. Funktion "Vor markiertem Feld" bzw. "Am Schluss"). Da die Feldreihenfolge Eingabefelder-Ausgabefelder (siehe Definition) höchste Priorität hat, wird als Einfügefeld die nächste passende Stelle genommen, wenn der Typ nicht gleich ist. Beispielsweise wird ein Ausgabefeld vor dem ersten vorhandenen Ausgabefeld eingefügt, wenn als Einfügefeld ein Eingabefeld bezeichnet wurde.

Das neue Feld wird bei einem Namenskonflikt automatisch umnummeriert. Falls der Name des Feldes in der Felderliste bereits vorkommt, wird am Ende die Laufnummer 0 angehängt oder dessen Laufnummer um 1 erhöht, falls bereits eine enthalten ist. Dadurch kann man problemlos mehrere Felder des gleichen Typs in Serie einfügen.

Zum Schluss wird mit der Funktion "Legende zu mark. Feldern" auch eine Legende vor das neue Feld platziert. Anders als beim Direktauslösen dieser Funktion gilt jedoch: Falls die Legende ein anderes Objekt zu mehr als 3/4 überlagern würde, wird sie ganz weggelassen. Das ermöglicht das einfache Platzieren mehrerer Felder horizontal nebeneinander. Falls gewünscht kann man auch zum einem späteren Zeitpunkt mit obiger Funktion die Legende generieren lassen.

Am Ende der Platzierung wird der Einfügepunkt automatisch nach unten auf die nächste Zeile der Gitters gesetzt, damit man mehrere Felder vertikal untereinander anordnen kann.

# Die Funktion "Vor markiertem Feld"

Die Funktion legt das Einfügefeld fest. Das Einfügefeld ist das Feld, vor dem ein Feld aus der Stehenden Lupe (erzeugt mit der Funktion "Attr- Feldliste öffnen") in die Felderliste eingefügt wird.

Bei Layouts hat die Reihenfolge Eingabefelder, dann Ausgabefelder höchste Priorität. Wenn also das eingefügte Feld einen anderen Typ hat als das markierte Feld, wird so nahe wie möglich an das markierte Feld herangegangen. Die Reihenfolge wird aber nicht verletzt.

# Die Funktion "Am Schluss"

Die Funktion legt das Einfügefeld auf das Ende der Felderliste fest. Ein aus der Stehenden Lupe der Funktion "Attr-Feldliste öffnen" eingefügtes Felde wird am Schluss der Felderliste angehängt.

Bei Layouts hat die Reihenfolge Eingabefelder - Ausgabefelder höchste Priorität. Beim Platzieren eines Eingabefeldes heisst dies, dass es in jedem Fall vor dem ersten Ausgabefeld eingeschoben wird.

# Die Funktion "Default-Werte"

Diese Einstellung legt fest, ob eine Bearbeitungsfunktion im unteren Teil des Menus auf alle Zeilenarten eines Formulars wirken soll. Dabei spielt es keine Rolle, ob die Zeilenart in einem Fenster geöffnet ist oder nicht.

# Die Funktion "Koordinatenart ändern"

Der EMBASSY-Layoutgenerator erzeugt prinzipiell nur Textuelle_Layouts und Zeilenarten. Eine solche Zeilenart kann nur Felder und Legenden speichern. Sie haben Koordinaten und Ausmasse in der Masseinheit "Zeichen". Für den Ausbau der Zeilenarten mit grafischen Objekten muss eine Zeilenart umgewandelt werden, sodass sie Koordinaten und Ausmasse in realen Ausmassen hat.

Als Prinzip gilt: Alle Zeilenarten eines Layouts müssen gleichzeitig umgewandelt werden. Danach werden alle Einheiten in Bruchteilen von Zentimetern (Units) angegeben. Ein solches Layout nennt man Grafisches_Layout.

Die Umwandlung funktioniert so, dass die Zeilenart mit einem Gitter von den durchschnittlichen Ausmassen des Layout-Grundformats überzogen wird. Alle Koordinaten und Ausmasse werden sodann auf dieses Gitter umgerechnet.

Während der Umwandlung werden alle Grundformate in ihre grafischen Entsprechungen umgewandelt. Beispiel: Aus "XA Textdruck Normale Schrift 0" wird "XR Grafikdruck Normale Schrift 0" usw.

Damit man nicht alle mit Grafikzeichen erstellten Ränder und Trennlinien wieder neu zeichnen muss, erkennt die Umwandlungsroutine diese und wandelt sie automatisch in Grafikstriche um. Die Grafikstriche bekommen als Eigenschaften die Einstellungen wie unter der Funktion "Einfügen->Default-Werte" angegeben.

Für die Papiereinstellungen wird A4 hoch mit den meistgebräuchlichen Rändern schon mal eingesetzt. Falls jedoch mind. eine Zeilenart >= 21 cm breit ist, wird A4 quer eingesetzt und der linke und obere Rand vertauscht.

Befindet sich das textuelle Layout in einem TDF, kann man direkt den zweiten Schritt anhängen und mit der Funktion "Datei->Speichern unter..." das Ziel-GDF produzieren.

# Die Funktion "Legenden definieren"

Mit dieser Funktion können neue Legenden erzeugt werden. Legenden sind Textstücke, die entweder linksbündig, rechtsbündig oder zentriert in einem Rechteck angezeigt werden. Bei der Form der Expandierten_Legende beginnt das Textstück linksbündig, das letzte Zeichen wird rechtsbündig platziert und dazwischen mit einer feinen Punktreihe aufgefüllt.

Zuerst wird mit dem Pointer das Rechteck für die Legende aufgezogen. Die Höhe wird automatisch auf die Höhe des Textes reduziert. Bei eingeschaltetem Gitter wird die Legende vertikal im Gitter zentriert.

Weil es praktisch ist, werden die nicht-expandierten Legenden automatisch mit der Funktion "Ausmasse auf Standard" nachbearbeitet, sodass das Rechteck bündig ist. Falls man dies nicht wünscht (selten), kann man es mit der Funktion "Zurück" aus dem Menu "Bearbeiten" ungeschehen machen. Die Legende hat dann wieder die im Info-Dialog eingegebenen Ausmasse.

# Die Funktion "Rahmen/Trennlinie def."

Rahmen sind dreidimensional aussehende Objekte zur optischen Abtrennung von anderen Objekten. Ein Rahmen ist sehr vielgestaltig:

  1. Im Normalfall ist es ein simples Rechteck. Falls eine Rahmenlegende definiert ist, wird sie im linken oberen Bereich in den Rahmen eingefügt.

  2. Falls die Höhe weniger als zwei Textzeilen ist, wird der Rahmen auf eine ##Horizontale_Trennlinie reduziert. Eine eventuelle Legende wird ganz links am Anfang eingefügt. Ohne Legende geht die Linie ganz durch.

  3. Falls die Breite weniger als 1 cm ist, wird er auf eine

Vertikale_Trennlinie reduziert. Eine eventuelle Legende wird ignoriert.

Neue Rahmen können auf zwei Arten gemacht werden: Zum einen mit dem Modus "Rahmen/Trennlinie definieren", zum anderen mit der Funktion "Einfügen->Rahmen um mark. Objekte".

Rahmen sind zusätzlich ein wichtiges Gruppierungselement für Felder. Felder innerhalb eines Rahmens können automatisch markiert oder deren Feldnummern aufsteigend angeordnet werden.

Es bringt Vorteile, wenn man immer einen Rahmen um eine Gruppe von Eingabefeldern macht, auch wenn die Felder visuell nicht umrahmt sein sollen. Dazu gibt es die Möglichkeit des Unsichtbaren_Rahmens, der beim Design als gepunktete Linie erscheint. Dabei wird einfach das Flag "Unsichtbar" im Rahmeninfo eingeschaltet (siehe Funktion "Unsichtbar machen").

# Die Funktion "Linien definieren"

In diesem Modus können neue Linien ins Layout eingefügt werden. Bei eingeschaltetem Gitter wird sowohl der Anfangs- wie Endpunkt auf die Mitte der Gittermaschen eingerastet.

Hinweis: Um moderne, dreidimensional aussehende Trennstriche zu machen benutze man nicht diese Funktion, sondern die Funktionen den Modus "Rahmen/Trennlinie def."

Durch Halten der Taste [Shift] während dem Ziehen kann erreicht werden, dass die Linie genau horizontal, vertikal oder diagnoal in einem Winkel von 45 Grad wird.

Durch Halten der Taste [Alt] wird der Gitter-Einschaltzustand zwischenzeitlich umgeschaltet.

# Die Funktion "Rechteck/Quadrat definieren"

Mit diesem Modus werden neue Recktecke oder Quadrate ins Layout eingefügt. Bei eingeschaltetem Gitter wird die linke obere und rechte untere Ecke auf die Mitte der Gittermaschen eingerastet.

Hinweis: Um moderne, dreidimensional aussehende Rahmen um eine Reihe von Eingabefeldern zu machen benutze man nicht diese Funktion, sondern die Funktionen "Einfügen->Rahmen um mark. Objekte" oder den Modus "Rahmen/Trennlinie def."

Durch Halten der Taste [Shift] während dem Ziehen kann erreicht werden, dass ein Quadrat entsteht.

Durch Halten der Taste [Alt] wird der Gitter-Einschaltzustand zwischenzeitlich umgeschaltet.

# Die Funktion "Ellipse/Kreis definieren"

Mit diesem Modus werden neue Ellipsen oder Kreise ins Layout eingefügt. Bei eingeschaltetem Gitter wird die linke obere und rechte untere Ecke der aufgezogenen Rechtecks auf die Mitte der Gittermaschen eingerastet.

Durch Halten der Taste [Shift] während dem Ziehen kann erreicht werden, dass ein Kreis entsteht.

Durch Halten der Taste [Alt] wird der Gitter-Einschaltzustand zwischenzeitlich umgeschaltet.

# Die Funktion "Felder definieren"

Mit dieser Funktion können neue Felder ins Layout eingefügt werden. Nach der Festlegung der Ausmasse erscheint eine Lupe zur Auswahl eines Feldes, vor dem das neue Feld eingefügt werden soll.

Beim grafischen Layout und eingeschaltetem Gitter wird das Feld vertikal in der Gitterzelle zentriert und horizontal um den Platz des Felddelimiters verbreitert. Die Verbreiterung hängt vom Layout-Typ ab, dh. ob es sich um eine Eingabemaske, Ausgabemaske oder Druckformular handelt. Da der Typ Ein/Ausgabemaske beim Definieren des ersten Feldes noch unklar ist, wird der bisherige Typ verwendet. Wenn man also eine Eingabemaske leermacht um eine Ausgabemaske zu designen, muss man zumindest die Layout-Informationen ändern oder die Maske speichern und neu laden.

Falls das Feld im Nummernbereich der Ausgabefelder liegt, wird ein Ausgabefeld vorgeschlagen, sonst ein Eingabefeld. Der Trennstrich muss hier ausgewählt werden, wenn man ein Eingabefeld hinter dem letzten einfügen will. Bei Auswahl des ersten Ausgabefeldes wird angenommen, es soll ein Ausgabefeld vor dem ersten eingefügt werden.

Die Feldnummer bestimmt bei einer Maske die Reihenfolge, in der die Felder eingegeben werden. Um eine Gruppe von Feldern zu einer Eingabeeinheit zu gruppieren, umgibt man sie am besten mit einem (ev. unsichtbaren) Rahmen. Weiteres siehe bei Funktion "Rahmen/Trennlinie def.".

Falls man beim Einfügen das Flag "Ausgabefeld" umschaltet, wird nach der Bearbeitung automatisch die Funktion "In Eingabefeld umwandeln" bzw. "In Ausgabefeld umwandeln" aktiviert. Dadurch wird zwar die Einfügeposition wieder zerstört, dafür bleibt die Gruppierung erhalten.

Der Einfüge-Punkt wird wie beim Einfügen von Feldern aus einer Felder- oder Attributsliste unter das soeben platzierte Feld verschoben, damit man gegebenenfalls mit dieser Funktion weiterarbeiten kann.

Tipp: Nach dem Einfügen kann man mit der Funktion "Einfügen->Legende zu mark. Feldern" aus dem Kommentar die Legende dazu generieren.

# Test-Formular für die Papierzufuhr

Zum Austesten der Papierzufuhr gibt es im Programm ein eigenes grafisches Test-Formular (Programmdatenfile "PDPLA00.PDF"). Dieses kann man mit der Funktion "Test-Formular" laden.

Das Formular enthält für jede im EMBASSY definierte Papierzufuhr (gemäss Dokument 'PaperSources') eine Zeilenart mit Grösse ca. 16 x 16 cm. Dies ist so gross, dass eine Zeilenart alleine auf eine Seite geht, aber nicht 2 davon zusammen. Dies ist wichtig, da sonst diejenige Zeilenart, welche weiter unten kommt, die Papierzufuhr der oberen übersteuert.

Im Text der Zeilenart steht, wie die Papierzufuhr heisst gemäss Dokument 'PaperSources', ob es sich um einen von Windows vordefinierten Standard- Schacht handelt und die Nummer, mit der man im Windows den Schacht ansteuert. Siehe dazu auch den Text "Printing.TXT" Kapitel "Papierschächte".

# Ausprobieren

Mit der Funktion "Testdruck" aus dem Menu "Layout" kann man auswählen, welche Zeilenart man drucken will. Wenn man in jeden vorhandenen Schacht des Druckers eine andere Papiersorte (markiert oder farbig) einlegt, sieht man nachher daran, woher der Drucker das Papier genommen hat.

# Probleme mit der Papierzufuhr

Der Grund der Probleme ist, dass Windows Papierschächte per Nummer angesprochen werden, Windows aber nur eine kleine Menge an Papierschächten fix vordefiniert. Einem vielseitigen Drucker reicht dies in der Regel nicht, weshalb er eigene Schachtnummern ab 256 besitzt. Diese stehen häufig nicht im Manual, oder sind falsch aufgeführt. Ein einmaliger Test ist dann unausweichlich.

Wenn das Papier nicht vom richtigen Schacht genommen wird, kann dies verschiedene Ursachen haben:

  • falscher Druckertreiber
  • fehlerhafter Druckertreiber
  • ungünstige Standard-Einstellungen am Drucker selber
  • Fixe Einstellung des Papierschachtes beim Druckerobjekt im Druckerordner. Beim HP LaserJet 4 beispielsweise Schacht-Sperren oder "Zuerst"-Einstellung überprüfen.
  • Falsche Angaben im EMBASSY-Dokument "PrinterBins" im "PEMPA98.PDF".

Mit dem Megos-internen Testprogramm "TPrinters.EXE" und "Druckereigenschaften" kann man übrigens auch ausprobieren, welcher Schacht (gemäss Druckertreiber) welche Nummer bei Windows hat.

  • fraglichen Drucker als Standard setzen
  • im Eigenschaften (resp. Dokument-Eigenschaften) einen Schacht als Standard setzen
  • in 'TPrinters' mit Eingabe "51" die Werte anzeigen lassen, die unterste Nummer ist die Schachtnummer.
  • dann die anderen Schächte der Reihen nach setzen und Eigenschaften speichern, dann mit "51" wieder anzeigen lassen.

# Dokumentation der Papierzufuhr

Die Excel-Tabelle "PaperSrc.XLS" enthält die Zuordungen zwischen den verschiedenen Nummerierungen und Begriffen:

  • Windows-Nr. und Name ("WinGDI.H" resp. "ODevices.DEF")
  • EMBASSY-Aufzählungs-Typ mit ORD-Wert ("MSystem.DEF")
  • Dokument 'PaperSources' mit Wahlzeichen und Bezeichnung.
