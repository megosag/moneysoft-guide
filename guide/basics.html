# Basics

# Screen Overview

The screen, or more exactly the main window of any Moneysoft application, basically has the following 4 regions:

  1. Main menu bar (menus with various configuration functions and options)
  2. Toolbar (shortcuts to frequently used functions)
  3. Status bar (current program, current client, current dialog mode)
  4. Use area (program dependent, here in the launcher program groups and program lists)

# Installation Overview

The Installation Overview as displayed by the EMBASSY Launcher shows the list of all installed programs of all applications. It's a tree view with folders that you navigate like folders in the Windows Explorer.

# Organizer

Beside the installation overview the launcher also offers a window called Organizer that you can use to configure a list of your personal favorites. (All launcher windows have a menu Launcher with a function Show Organizer to bring up this window if its not currently open.)

You can copy applications, program groups and individual programs from the installation overview to the organizer by using Drag&Drop:

You create new own folders using the function New Folder in the Functions menu. You can open such folders in separate windows by double-clicking their entries in the organizer.

You can use Drag&Drop to modify the display order of the programs.

With the function New Link... in the menu Functions you can create organizer entries for arbitrary external programs and files.

You delete any organizer object with the Delete function in the menu Functions or with the Delete key.

The function Save State in the menu Launcher saves all your changes. (The launcher configuration is user-specific; it's also auto-saved when you close the Launcher.)

# Lenses

Lens is the name of a general Moneysoft user interface element that is used whenever there are lists or tables of things to display, with the possibility to select one. (In Windows, something like that is usually called List View.). The lens comes in two varieties, as main lens, simply called Lens and available through the F3 key, and as Additional Lens, available through the Shift+F4 key combination.

Lenses are typically available on input fields in dialogs where they show the set of possible values for the field:

A field with a main lens displays an arrow icon to indicate the availability of the lens: Such a field more or less works like a standard Windows combo box, and you can bring up the lens either by pressing the already mentioned F3 key or by clicking on the arrow icon with the mouse.

A field can offer Lens and Additional Lens at the same time which results in two little icons, one on top of the other: On such a field, F3 and Shift+F4 are active and both icons available for clicking.

You select an entry of a lens by using the Enter key or by double-clicking it with the mouse.

There is a way to search within all the data that a lens displays, regardless of the type of data, by using the function Find... in the menu Lens (also available through the Ctrl+F key combination). You enter the text to search in the dialog box that appears:

Without that asterisk search will only include data in the key column, the column that is drawn with a darker yellow than the other columns.

Search starts as soon as you hit Enter on the search field. Results will be shown in a new lens:

If you select an entry in the result lens it will be marked in the original lens where it is then ready to be selected and returned to the point where the original lens was opened.

# Find Securities

Whenever the cursor is on a field for a securities number the function Browse... in the menu Help (also available through the Shift+F1 key combination) is active and starts the program Find Securities. This program lets you search for securities using several methods and finally gives back the number of a particular security to the field where you started after you select it the result lens.

On fields like the one for the name you can give a short text to find all securities whose names start with that text: E.g. "Future" will match a security named "FUTURE DAX PERF JUN10". By starting the text with an asterisk * you can select all securities that contain the given text anywhere within the name: E.g. "*gold" will match "ANGLOGOLD ASHANTI".

Giving values on more than one field works like logical AND.

# Global Field History

The Moneysoft programs offer a feature called global field history that facilitates entering values into fields in cases where identical values occur repeatedly for different records or for different runs of a program, which depending on the data happens quite frequently: Behind each and every input field of every dialog is a history of the 10 last distinct values entered or displayed. This history even survives the termination and subsequent restart of a program.

There are four functions to work with the history, listed at the end of the menu Edit. All of them have keyboard shortcuts, and it's worth the effort to memorize those because using them ist much faster than navigating to the menu functions.

The most basic history function is Field Value List, with a keyboard shortcut of Ctrl+H (press first the Ctrl key, then add the H key): This brings up a lens with the history of the field values where you can select one to set the field. Of course the letter H was chosen for this function because the word "history" starts with it.

The function Previous Value, with a keyboard shortcut of Ctrl+U (with "U" for "up"), browses back (or up) through the values in the history while you stay in the field.

The function Current Value, with a keyboard shortcut of Ctrl+R (with "R" for "return") restores the field value as it was before you startet to change it by using Ctrl+U.

The function Value List, with a keyboard shortcut of Ctrl+K, shows a lens of historical values, but unlike Ctrl+H the values come out of the histories of all fields with the same type and length as the current one: If you enter a securities number on a field in one program, Ctrl+K will include that number on another field for a securities number in another program.

# Modes

# Delete Mode

If you actively want to delete something in Moneysoft yourself, in almost all cases you have to go through the following process which is deliberately multi-step to avoid accidental deletions:

As step 1 you bring up the record to delete in a dialog so that the dialog goes into change mode, as indicated by the word Change in the the status bar. (It does not matter how you get the record, by entering keys manually, by selecting it in a lens, or by browsing through the records.)

Now you have to switch the dialog to delete mode. The simplest way to do so is using the corresponding "X" icon in the toolbar (step 2).

Switching the mode places the cursor to the Input ok? field. Only if you confirm here by entering "Y" for "Yes" or by clicking the OK button the data record is finally deleted. By entering "N" for "No" you can back off, and nothing is deleted.

The following error message appears if you click on the Delete Mode icon in the toolbar while the dialog is not ready i.e. is not in the right mode:

This may happen to people new to Moneysoft on dialogs with more than one key field: Only after you enter the last of the key fields the program can read the corresponding record, display it and switch the dialog to change mode. (If you are unsure about which fields are key fields closely watch the space for displaying the mode in the status bar: As long as nothing is written there, the mode is still undefined because you are still on a key field.)

# Macros

# The Case For Macros

Macros in Moneysoft are recorded key presses, function activations and button clicks. With a macro you can execute a multi-step process several times in a very rapid and easy way. Macros are especially useful for mass mutations of data, e.g. setting the stock market of several dozens of securities to a specific (identical) value.

You start recording a macro by clicking the corresponding "red dot" icon in the toolbar: Everything that you do now in the current program goes into the macro. You stop the recording by clicking the "red crossed" icon in the toolbar.

You can use macros in an ad hoc fashion, letting them vanish as soon as you leave the program, or you can name and store them for future re-use, as explained in the next chapter.

Caution

When recording and storing macros try to avoid using mouse clicks. A macro with recorded mouse clicks may only play correctly if everything involved (windows, dialogs, buttons) is still at the exact same screen coordinate as of the time of recording which of course is not always the case.

# Managing Macros

The function Macros... in the Support menu brings up a lens of all stored macros where you can select one for execution:

You can use this lens also to store a new macro that you just recorded, with the Save... function in the Macro menu:

For storing you have to give it a name, and you can optionally store a description with it. It is highly recommended to use expressive names and detailed descriptions, as macros because of their static nature are highly specific to circumstances, and a macro executed in a wrong context or at the wrong time may more or less "run amok".

A macro with the Dialog-specific? checkbox ticked is visible only on the field where its recording started. If you bring up the lens with the stored macros on that field later, those field-specific macros are listed first which helps you keep the overview. (If you do want to see all macros, use the Show All function in the Macro menu.)

# Options

# Selecting the Report Language

In a program that prints reports you can specify the language to use for a report, for each specification separately, as part of the standard options.

There are 4 different language selection modes available:

  • User language
  • Client language
  • System language
  • Specification language

The user language is the language of the user that runs the report program: Using this selection mode, the language of the report thus depends on the current user.

The client language is the language of the current client at the time of the program run.

The system language is the globally defined language for the Moneysoft installation as a whole, and thus user-independent and client-independent.

The specification language is the language contained in the particular specification that is used to print the report, given as part of the standard options on the field immediately following the language selection mode field.

# Language Defaults

There are many possible situations where a language is undefined, or something (a dialog, an error message, a report, a name of something like a security) is not available in the desired language.

For such situations Moneysoft implements a language default mechanism that works with the following 3 levels of descending priority:

  1. User language
  2. System language
  3. German

In the context of a language selection mode for a report this would work like this: "If the user language is selected but undefined, check whether there is a system language defined. If yes, take that instead. If the system language is undefined as well, default to German."

In the context of displaying e.g. a dialog it's basically the same: "If the particular dialog is translated into the current user's language, display it in that language. If not, check whether it's available in the system language. If not, display the dialog in German, as every dialog is available in German."

Some basic data is also multi-language: E.g. a security can have up to 4 names, in the 4 supported languages German, English, French and Italian. Because there must be data for at least one language (there can't be any nameless securities) the programs force you to specify data at least in the system language; the fields for the other languages can be left empty, however.

# Specification Descriptions

A specification in a report program is identified by a name: That name is the direct key used to select it.

In a multi-language environment this can be a problem because there is only one name for each specification. If you give your specifications "real" i.e. descriptive names you face the difficulty that they will be in a single language only.

There are however 4 specification descriptions for 4 languages as part of the standard options. After selecting a specification its description in the user language is displayed immediately below its name.

So if you work with several languages it's a good idea to simply use numbers as names for your specifications and fill the descriptions in as many languages as desired. (The 3- level language default mechanism described in an earlier chapter also works for descriptions.)

(Note: If you upgrade from an earlier Moneysoft release to release Amber which is the first one to support specification descriptions specifications will not be numbered automatically like recommended here; they will keep their names. If you want them numbered, you must delete and re-create them manually.)

# Form Variants

Every program that prints reports has an object called form that specifies the visual aspects of the report. Such a form typically exists in more than more variant because different variants of the report are needed:

A report with data about securities is at one time needed as a list with one line per security containing only key information, printed in portrait mode (variant 0). At another time you want the same report about the same securities, but listing detailed info per security on multiple lines, printed in landscape mode (variant 1).

Languages add one more dimension to this: One form variant for the "summary list" of securities is not enough, as you may want to print this list in different languages. Because a single form variant cannot be multi-language in itself, one variant per language is needed.

The possible number range for variants is from 0000 to 9999. The following sub-ranges are defined for variants in different languages:

  • 0000 – 0999: German
  • 1000 – 1999: English
  • 2000 – 2999: French
  • 3000 – 3999: Italian

Now, if the "principal" variant 1 is the "detailed list" from the example above, as per convention variant 0001 is that list in German, variant 1001 that list in English, and 2001 that list in French.

All the standard form variants that come with Moneysoft follow this variant numbering convention. If you design your own additional variants you should follow this convention because the report program will "calculate" the variant number by using the language selection mode; if the variant numbers do not follow the convention, this won't work i.e. will not reliably print reports in the desired language.

Missing language variants are allowed: If there is no variant for the language as per selection mode Moneysoft will use its 3-level language default mechanism explained earlier to automatically switch to a variant in another language.
