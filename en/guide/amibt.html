# Banken-Interface

Transaktionen, Devisentermingeschäfte und Cashbuchungen können mittels Schnittstelle ins Moneysoft übernommen werden. Dabei gibt es verschiedene Liefermöglichkeiten, entweder direkt in die Importschicht in unserem Format oder wir transformieren die gelieferten Files via Mappings in unser Format.

# Mapping-Tabellen

Wenn andere Systeme die Transformation der Lieferung in unser Format übernehmen wollen, braucht es gewisse Mapping Tables als Lookup für die Lieferung der korrekten Werte in unser Format. Dafür gibt es 4 Lookup Tabellen:

  • Mandanten
  • Depotstellen
  • Kontokorrent-Konten
  • Titelstamm

In diesen Tabellen findet man alle Stammdaten die benötigt werden, damit eine Zusweisung zu unseren Keys gemacht werden können. Das Programm Mapping-Tabellen schreiben ist zuständig für die Nachführung der Mappingtabellen. Diese können entweder via Batchfile in einem Scheduler regelmässig oder bei Bedarf manuell nachgeführt werden.

# Titel-Neueröffnungen

Es besteht die Möglichkeit fehlende Titel in der Titelstamm Mapping Tabelle als zu eröffnende Titel zu melden.

Wenn neue Titel eröffnet werden müssen, können diese mit dem Programm Titel-Neueröffnungen ausführen aufgerufen werden. Ist die apiD-Schnittstelle zur SIX aktiv, lassen sich diese mit einem Klick via SIX eröffnen, ansonsten wird der Header gem. den gelieferten Infos geschrieben und die restlichen Informationen müssen noch ergänzt werden.

Der Titel wird automatisch auf den Status Eröffnet gesetzt und entsprechend eingefärbt, sobald dieser eröffnet wurde und verschwindet beim nächsten Programmstart von der Lupe.

# Titel bereits vorhanden

Ist der gemeldete Titel bereits im Titelstamm vorhanden, erscheint die Meldung, dass der Titel bereits existiert und einige Angaben mit dem vorhandenen Titel verglichen werden. Sind Angaben abweichend wird dies ebenfalls in einer Dialog-Box gemeldet.

Abschliessend wird in einer Dialog-Box gefragt, ob der Status des zu eröffnenden Titels auf Eröffnet gesetzt werden soll.

# Daten importieren

Sobald die Transaktionen transformiert und bereit zum Import stehen, können diese mit dem Programm Daten importieren importiert werden.

Unter Selektionen lassen sich verschiedene Selektionen durchführen um nur einzelne Trades oder Buchungen zu importieren, z.B. von einzelnenen oder mehreren Clients, Konten oder Depots oder das Eingrenzen des Datums.

Mit dem Button Lupe Import-Records lassen sich die zu importierende Records zeigen oder ausblenden. Die Anzeige der Records wird dynamisch anhand der Selektion gefiltert.

Mittels dem Button Unvollständige sichtbar können Transaktionen angezeigt werden, die noch nicht vollständig mit den nötigen Informationen ausgestattet sind. Diese Records können nicht importiert werden und dienen lediglich der Vorinformation.

Mit bestätigen des OK Buttons beginnt die Verarbeitung. Während der Verarbeitung werden die Daten nochmals auf Vollständigkeit geprüft und validiert.

# Optionen

Unter Optionen lässt sich zusätzlich selektieren welche Daten importiert werden sollen. Folgende Auswahl steht zur Verfügung:

  • Transaktionen
  • Cash-Buchungen
  • Devisentermingeschäfte
  • Corporate Actions

# Fehlerhandling

Werden Fehler während der Verarbeitung festgestellt, werden diese in einem umfangreichen Debug-Protokoll mit Debug-Outputs und zusätzlich einem leserfreundlicheren Quick-Protokoll mit den wichtigsten Infos geschrieben.

Das Quick-Protkoll wird direkt nach dem Import geöffnet, wenn die Option Protokollfile anzeigen? gesetzt ist.

Fehlerhafte Records können anschliessend analysiert und die Parameter neu eingestellt und danach den Status von Fehler auf Bereit zum Import direkt in der Übersicht zurückgestellt werden. Ebenfalls lässt sich der Status auf Manuell erledigt stellen, falls es sich um einen Ausnahmefehler hält und keine Parametrierung notwendig ist. Beim Ändern des Status kann ein Kommentar hinzugefügt werden.

# Verbuchung

Die Importierten Records werden alle ins Journal gestellt wo sie noch definitiv verbucht werden müssen. Beim Verbuchen werden wieder verschiedene Validierungen gemacht um sicher zu stellen, dass alles korrekt verarbeitet wird.

Solange ein Record im Journal steht, können diese noch manuell bearbeitet werden.

# Positionsabstimmung

Mit dem Programm Positionsabstimmung anzeigen/schreiben werden die Daten von Moneysoft mit den gelieferten Daten der Gegenpartei verglichen. Dazu können von der Gegenpartei die Daten direkt in die Tabelle in unserem Format geschrieben werden oder via File geliefert werden, welches bei uns transformiert wird.

Wichtig

In der Übersicht wird einerseits die Menge, der Ratazins und der Marktwert verglichen. Die Menge muss zu 100% übereinstimmen, ansonsten generiert es einen Fehler.

Für die anderen zwei Abweichungen können pro Titelart die prozentualen Grenzen für den Status grün, orange und rot im Parameter-Dokument Abt/AbwTitelart im File D000Vrdk.DAT hinterlegt werden.

Die berechneten Vergleiche werden Revisionssicher in einer Tabelle gespeichert und lassen sich jederzeit wieder aufrufen.

Die Anzeige lässt sich über den Standardmenupunkt Lupe – Exportieren leicht ins Excel exportieren, dabei wird die Formatierung beibehalten.

# Regeln Steuerbeträge

Mit dem Programm Regeln Steuerbeträge verwalten können Regeln zur Berechnungsgrundlage bestimmt werden.

Die Regel kann sowohl auf Mandant-Ebene oder global ohne Mandant definiert werden.

Mit der Reihenfolge-Nummer kann die Reihenfolge manuell bestimmt werden, wie die Regeln abgearbeitet werden. Die Mandant-Spezifischen greifen immer vor den Globalen.

Bei folgenden Felder können einzelne Werte kommagetrennt (auch mit Wildcards), wie auch Bereiche hinterlegt werden:

  • Domizile Mandant
  • Domizile Titel
  • Nationalitäten Titel
  • Titelarten
  • Valorennummer (ISIN)
  • Depostellen

Unter Berechnung kann gewählt werden ob es die Regel von Moneysoft anwenden soll, oder das übernimmt wie die Bank es liefert. Wenn keine Regel definiert wurde, ist Standard wie die Bank es liefert. Hier kann aber mit der Reihenfolge noch eine explizite Regel vorgeschoben werden, wenn ein spezifisches Depot doch von der Bank kommen soll z.B. und alle anderen nach Moneysoft.

Unter Gültig ab kann bestimmt werden, für welche Trade-Dates die Regel gültig ist.

Unter Absolute Toleranz (Betrag) und Relative Toleranz (Prozent) kann die gültige Abweichung definiert werden.

Info

Die Rückforderungssätze können unter Valor - Mandanten - Rückforderungssätze verwalten eingestellt werden.

# Weitere Parametrierungen

# Kontokorrent Konten

Im Programm Valor - Stammdaten - Kontokorrent-Konten verwalten müssen die Bank-Kontonummer und die Kontonummer vom Final erfasst werden. Diese werden für das Mapping der angelieferten Files benötigt.

# Parameter

Im Dokument Bip/BankIntParam im File D000Vrdk.DAT können Grundeinstellungen vorgenommen werden wie:

  • Verzeichnis Reports
  • CB: Durchlauf-Konten
  • Einstellungen für DTGs
