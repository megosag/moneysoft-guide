# Final / Financial Accounting

The icons for the program groups and programs of Final in the launcher use the color green to make them easily recognizable as belonging to Final.

# Clients

# Managing Clients

Clients in Moneysoft and thus in Final are identified with numbers in the range 000 to 999.

You manage the clients with the program Manage Clients in program group Clients. The most important parameters of each client are the following:

Account number pattern defines the length of account numbers and the characters that are valid in such numbers. Each account number entered in any Final program is checked against this pattern.

Base currency is defined once when creating a new client. As soon as there are posted entries for the client you cannot change the base currency anymore.

Start of fiscal year must be set initially but is later managed automatically by Final i.e. the year incremented during fiscal year change.

Subject to VAT? must be checked for all clients that are liable/subject to VAT. With this option set tax/VAT codes are applied when creating journal entries.

Final can automatically generate voucher numbers with automatic increment for you. It's possible to specify an initial value in the field Last voucher number.

# Change Client

For each user separately, one of the existing Final clients is the current client, and all Final programs automatically use the client that is current at the time of the start of the program. The status bar shows you which client is the current one.

You can switch between clients with the program Change Client in the program group Clients.

Important

This program (like its sister programs in Valor and Emcal) switches the current client for Final, Valor and Emcal (if installed) simultaneously to keep them in sync. However, all programs that are already running i.e. have windows open do not change the client. To work with the new client you have to terminate and then restart those programs.

# Chart of Accounts

# Manage Chart of Accounts

Each Final client has ist own chart of accounts. You manage that with the program Manage Chart of Accounts in the program group Chart of Accounts.

If a client is linked to a master client you can only make changes to the chart of accounts in the master client but not in the dependent client itself.

The values of the field in the Reports / counters frame specify how totals are calculated in reports. Balances go into totals in the order that is defined by account numbers. Each profit account balance and each balance sheet account go into a particular counter. Typically a total account takes its value (its "balance") from a counter. You can use also counters to add several totals to a total of a higher order; you can build elaborate account classification systems this way.

You can link an account of type Title with a total account by using the same counter for both.

You can tell Final to suppress unnecessary titles in reports by setting the option Print balance of 0 in the frame Reports / layout to the code for If entries present.

In the frame Interest you can specify debit and credit interest rates. Those interest rates will be valid globally for that account. If you want dynamic and changing interest rates you have to work with interest postings.

The parameters in the Reports / layout frame influence reports with charts of accounts. An account can have a second balance in a so-called auxiliary column. Where the normal, "primary" balance keeps amounts in a given currency you typically use the auxiliary column to store unit values like number of shares or number of some kind of rights.

There is a special currency with code FWG (for "Fondswährung") that you can use in master charts of accounts. If you create a new dependent client Final will automatically replace FWG with the base currency of the new client.

The accounts for currency differences are used by the program Create Entries for Exchange Rate Gains in program group Fiscal Year Change.

The following values are mandatory i.e. needed for each account:

  • Number
  • Name (at least in the system language)
  • Type
  • Debit/credit code (strongly recommended to always set to Debit)
  • Consolidate (strongly recommended to set to "-" for no consolidation)
  • Auxiliary column (set to code Use for accounts where unit values can occur)
  • Currency
  • Accounts for currency differences (only if account currency differs from base currency)
  • Report / counters
  • Account level (use of code 0 recommended)
  • Print balance of 0 (use of code If entries present recommended)
  • Blank lines before
  • Blank lines after

# Interface Moneysoft Final to ABACUS

In a first program Manage Mapping Megos to ABACUS in program group Clients you manage the mappings i.e. specify which client, account or cost center in Moneysoft Final is linked to which client, account or cost center in ABACUS.

A mapping is always a mapping of one kind: You only ever give values in one particular frame, either for a client, an account or a cost center mapping.

For account numbers you can create mappings that are specific to a certain client or valid globally for all clients (as long as no client-specific mapping overrules); the same is possible for cost center mappings.

A second program Export Journal Entries to ABACUS exports posted entries from Final to ABACUS, based on the defined mappings.

Uncheck the option Export? to make trial runs without actually exporting i.e. writing an export file.

The program keeps track internally which entries were already exported. With option Only unexported entries? checked export is restricted to entries which so far were never exported. With the option unchecked all posted entries of the given fiscal year are exported.

# Account Assignment Assets/Liabilities

The program Account Assignment Assets/Liabilities in program group Reports lets you define the print order of asset and liability accounts in balance sheet reports in the following way:

The account given as Account from is printed at its normal position within the report if its balance is positive. If its balance is negative it gets a new position, behind the account given as Account to.

Important

Be aware that the account, in addition to moving within the report, also gets the counter information of the "to" account.

# Journal Entries

# Create Journal Entries

With the program Create Journal Entries in program group Journal Entries you can create new entries yourself and check entries that Valor generated based on transactions for securities.

This program works with numbers identifying the unposted journal entries simply called ID. These IDs are temporary and have nothing to do with the (permanent) journal entry numbers that are assigned when posting. When creating entries you can leave the ID field empty to let the program use the next available id.

For communicating with third parties about journal entries it is recommended to use voucher numbers; Final has a mechanism to automatically generate and assign sequential voucher numbers that do not depend on IDs or journal entry numbers.

If either the account or the contra account uses a currency different from the base currency the amount in that currency is needed as well. Final includes a list of exchange rates and proposes the corresponding rate here, but you are free to overwrite it if needed.

Unposted journal entries that were automatically generated by Valor have a corresponding origin displayed (top-middle of the dialog).

You can switch an entry from debit to credit or vica-versa by setting the field Debit/credit code accordingly, but you should not do so without good reason.

VAT values for new entries are set to defaults based on info in the charts of accounts about the two accounts involved; if needed, you can manually change/overwrite them.

Below the dialog is a lens showing all currently unposted entries. You can bring back one of them into the dialog for further changes by double-clicking it in this lens.

As long as entries are unposted you can freely modify or delete them.

# VAT

If you use an account with a tax/VAT code defined when creating journal entries and if the client is configured as subject to VAT in Manage Clients the program Create Journal Entries will automatically fill the fields in the frame VAT info. If necessary modify these default values before creating the entry.

VAT included? sagt ob der Betrag inklusiv oder exklusiv MWST eingegeben wird, dementsprechend wird der Net amount und der Taxable amount gerechnet.

Net? sagt ob die Netto- oder Brutto Abrechnungsmethode angewendet wird. Wird Brutto- Methode angewendet, wird beim Verbuchen keine MWST Buchung erstellt. Diese wird erstellt with the program Settle Value Added Tax in program group Journal Entries.

The program Settle Value Added Tax offers you the option to collect all VAT into a single journal entry per period, with the period's end date, by checking Collective booking?.

# Compound Entries

To create compound entries choose Compound entry for account or Compound entry for contra acc. as entry type in program Create Journal entries, depending on whether you want to create a credit or a debit item.

In a first dialog Common Information you specify the collective account, i.e. the contra account to be used, and the total of all individual entries. After confirming the first dialog you specify all the individual entries in a second dialog Individual Entries, giving a description and an amount for each one.

As you confirm one individual entry after the other the remaining balance is displayed in the first dialog left to the label Balance:.

To complete or abort this process leave the second dialog by pressing the ESC key or using the Cancel button. If the total of the individual entries does not match the amount given for the compound entry you will get the following dialog box Complete? to decide how to proceed:

With a "No" on Compound entry complete? and likewise "No" on Delete entries of compound entry? the whole process will simply resume i.e. you can just continue to enter more individual entries until their total does match the amount.

With "No" and "Yes" the program will delete all individual entries that you entered so far so that you can start over entering them.

With "Yes" and "Yes" you terminate the whole compound entry process with anything related to it deleted.

You can correct individual entries for compound entries the same way you correct "normal" journal entries, by selecting them in the lens Unposted Journal Entries or entering their ID manually on the corresponding input field. Correcting will take place in the Individual Entries dialog.

You can choose how the reports produced by the programs Print Statement of Account and Display Statement of Account will present compound entries:

Presentation depends on the value of the compound entry code for the corresponding account as specified in program Manage Charts of Accounts: With code Compound code only the collective account and the total amound will be shown. If you leave the code field empty all the individual journal entries will be shown:

# Cancelations

The program Create Journal Entries allows you to cancel posted journal entries, by going through the following steps:

  1. Click the Cancelation button or use the Cancelation function in the Functions menu to start the process
  2. A new dialog opens that lets you specify the posted journal entry to cancel
  3. If you know the journal entry number, specify that; alternatively, if you know the date, use that (shown in the screen shot below)
  4. Select the right entry in the lens to cancel it

If the selected entry is part of a compound entry all its entries will be canceled; it's not possible to cancel only a single entry out of a compound entry.

Do not cancel entries in this way that were created by Valor when posting a transaction. Cancel the transaction instead in Valor; this will automatically cancel all posted journal entries involved as well.

Important: Canceled posted journal entry are not deleted. To cancel, Final will generate a new entry that "negates" the other entry, and that new entry will be marked as a special cancelation entry. If you print lists of journal entries you will see both the original and the cancelation entry.

# Post Journal Entries

To make journal entries permanent you have to post them. You do so by using the program Post Journal Entries in program group Journal Entries.

With option Post? unchecked you can do a trial run and use the printed report to control what would happen if you did post.

# Reports

# Print Reports

The program Print Reports from program group Reports can be configured to print various reports, e.g. balance sheet reports.

If you only want posted journal entries within a period you fill both the "from" and the "to" Posting period fields. If you want a report as at a certain date, leave the "from" field empty and specify that date on the "to" field.

Important

Make sure the field Fiscal year has the correct value, i.e. the year that you want to report. The program proposes a default based on the current fiscal year as specified in program Manage Clients.

# Other Reports

Beside the generic Print Reports program there are number of other more specialized programs to print and display reports:

  • Print Clients' Report
  • Print Statement of Account
  • Display Statement of Account
  • Print Clients' Statement of Account
  • Print Clients' Account Balances

The "print" programs are all able to print to paper, to write PDF files and export directly to Microsoft Excel.

The program Display Statement of Account shows balance sheets in the form of a lens instead of printing them; this is handy for quick checks. You can still export those balance sheets to Excel by using the corresponding general lens function.

# Fiscal Year Change

The program group Fiscal Year Change contains a number of programs that are usually needed once a year to close one fiscal year in Final and start the next. Depending on reporting cycles, e.g. if you want to work with monthly valuations, you may use some of these programs also at various points during a fiscal year.

# Create Entries for Exchange Rate Gains

To value accounts with different currencies based on the current exchange rates you use the program Create Entries for Exchange Rate Gains in program group Fiscal Year Change: The program calculates a new balance in foreign currency, using the current exchange rate, and creates a journal entry for the difference to the old balance, using the accounts for currency differences as configured in the chart of accounts.

Like other such programs this one has an option as well to do trial runs: Set Create journal entries? to "N", and only the report will be printed, but not data created or modified.

Important

Use the selections that are part of the specifications to make sure only the appropriate accounts are processed. Depending on the accounting policy of the current client only short-term liabilities must be re-valued, but not long-term investments.

# Create Entries for Interest

The program Create Entries for Interest in program group Fiscal Year Change calculates interest for the selected accounts and for the time period specified. Interest rates to apply come either from the chart of accounts or from interest change journal entries.

# Start New Fiscal Year

The program Start New Fiscal Year in program group Fiscal Year Change makes the actual changes to start a new fiscal year.

There are the following preconditions for running this program: All closing entries are created, no new journal entries are needed, and all entries are posted – there are no unposted journal entries waiting.

The essential input fields of this program are:

Start of new fiscal year: The date of the start of the new fiscal year, which is checked against the start date of the current fiscal year in the client data to give a warning if the resulting period is not 12 calendar months.

Account number: The account to be used for the opening balance as at the first day of the new fiscal year.

After you confirm with "Y" on the Input ok? field or click the OK button the program generates opening entries for all balance sheet accounts and modifies the current fiscal year start date in the client data.
